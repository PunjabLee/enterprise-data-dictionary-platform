# 采集适配设计 v0.1

## 1. 设计目标

采集适配能力用于将分散在数据库、ETL、BI、接口平台、CMDB 和其他系统中的元数据汇聚到平台。首期重点是建立统一采集框架和差异识别机制。

## 2. 采集对象

| 对象 | 首期方式 |
| --- | --- |
| 系统清单 | Excel 导入，后续接 CMDB |
| 数据库元数据 | 直连只读采集 |
| 表字段 | 数据库系统视图采集 |
| 接口 | Excel/API 导入 |
| ETL/调度 | Excel/API 导入，后续适配工具 |
| 报表/指标 | Excel/API 导入，后续适配 BI |
| 标准术语 | Excel 导入和平台维护 |

## 3. 采集框架

每个采集器应包含：

- 连接配置。
- 采集范围。
- 采集任务。
- 原始快照。
- 标准化转换。
- 差异识别。
- 人工确认。
- 发布入库。

## 4. 差异类型

| 类型 | 说明 |
| --- | --- |
| 新增 | 源端出现新资产 |
| 修改 | 源端属性与平台已发布版本不一致 |
| 删除 | 源端资产消失 |
| 冲突 | 自动采集结果与人工治理定义冲突 |
| 缺失 | 必填治理属性缺失 |

## 5. 数据库采集

首期支持的能力：

- 数据库、Schema、表、字段。
- 字段类型、长度、精度、是否为空。
- 主键、唯一索引、普通索引。
- 表和字段注释。

采集限制：

- 只采集元数据，不读取业务明细数据。
- 连接账号必须只读。
- 敏感连接信息加密保存。

## 6. Excel 导入

导入要求：

- 模板固定版本。
- 支持预校验。
- 支持错误行导出。
- 支持分批导入。
- 支持导入批次追踪。
- 导入后进入待确认状态。

## 7. 开放 API

首期建议提供：

- 资产查询 API。
- 元数据导入 API。
- 血缘关系导入 API。
- 标准术语查询 API。
- 审批结果回写 API。

所有 API 必须鉴权并记录调用日志。
