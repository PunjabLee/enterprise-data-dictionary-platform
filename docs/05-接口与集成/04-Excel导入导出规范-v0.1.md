# Excel 导入导出规范 v0.1

## 1. 设计目标

Excel 导入导出用于首期资产盘点、数据补录、批量整改和验收交付。首期大量企业资产无法全部自动采集，因此 Excel 是必要入口。

## 2. 模板版本

每个模板必须有：

- templateType。
- templateVersion。
- sheetName。
- requiredColumns。
- enumColumns。
- referenceColumns。

模板版本写入隐藏或固定单元格，导入时必须校验。

## 3. 导入流程

```text
上传文件
-> 校验模板版本
-> 校验表头
-> 校验行数据
-> 生成导入批次
-> 写入临时数据
-> 生成错误明细
-> 成功数据进入待确认
```

## 4. 校验规则

| 校验 | 说明 |
| --- | --- |
| 必填校验 | 必填字段不能为空 |
| 枚举校验 | 状态、类型、等级必须在枚举内 |
| 唯一校验 | 编码、名称按规则唯一 |
| 引用校验 | 系统编码、业务域、责任人必须存在或可创建 |
| 格式校验 | 邮箱、日期、编码格式 |
| 长度校验 | 不超过字段最大长度 |
| 权限校验 | 上传人是否有导入范围权限 |

## 5. 重复处理

重复策略：

| 策略 | 说明 |
| --- | --- |
| reject | 发现重复直接报错 |
| update | 按编码更新已有对象 |
| skip | 跳过重复行 |
| manual_confirm | 进入待确认 |

默认策略：

- 核心资产：manual_confirm。
- 标准术语：reject。
- 字段字典补录：update。

## 6. 错误明细

错误明细必须包含：

- 行号。
- Sheet 名称。
- 字段名。
- 字段值。
- 错误码。
- 错误说明。
- 建议处理方式。

错误明细支持导出 Excel。

## 7. 导入批次状态

| 状态 | 说明 |
| --- | --- |
| uploaded | 已上传 |
| validating | 校验中 |
| valid_failed | 校验失败 |
| importing | 导入中 |
| success | 全部成功 |
| partial | 部分成功 |
| failed | 全部失败 |
| canceled | 已取消 |

## 8. 回滚策略

- 未确认数据可整批删除。
- 已确认但未发布数据可整批撤回。
- 已发布数据不得物理回滚，只能走变更或废弃。
- 每次导入必须记录批次和源文件。

## 9. 导出规范

导出类型：

- 当前查询结果导出。
- 模板下载。
- 错误明细导出。
- 影响分析报告导出。
- 验收清单导出。

约束：

- 导出必须校验权限。
- 敏感字段默认脱敏。
- 导出操作必须审计。
- 大批量导出走异步任务。
