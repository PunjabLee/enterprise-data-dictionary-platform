# 事件与消息集成设计 v0.1

## 1. 设计目标

事件与消息集成用于解耦平台内部模块，并为未来接入 RocketMQ、Kafka、外部通知系统、数据平台和治理运营系统预留能力。首期以本地领域事件为主，不强依赖外部 MQ。

## 2. 事件分层

| 类型 | 说明 |
| --- | --- |
| Domain Event | 领域内部业务事件，例如资产发布、标准发布 |
| Integration Event | 对外发布的集成事件，例如元数据变更通知 |
| Task Event | 异步任务事件，例如导入完成、采集完成 |
| Notification Event | 通知事件，例如待办提醒、整改通知 |

## 3. 事件抽象

统一端口：

```text
DomainEventPublisher
IntegrationEventPublisher
EventSubscriber
```

事件基础字段：

| 字段 | 说明 |
| --- | --- |
| eventId | 事件 ID |
| eventType | 事件类型 |
| aggregateType | 聚合类型 |
| aggregateId | 聚合 ID |
| occurredAt | 发生时间 |
| tenantId | 租户 ID，首期 default |
| traceId | 链路 ID |
| payload | 事件内容 |
| version | 事件版本 |

## 4. 关键事件

| 事件 | 说明 | 主要消费方 |
| --- | --- | --- |
| AssetPublished | 资产发布 | catalog、automation、notification |
| AssetChanged | 资产变更 | lineage、impact、notification |
| BusinessTermPublished | 业务术语发布 | dictionary、modeling、automation |
| BusinessTermDeprecated | 业务术语废止 | automation、governance |
| StandardPublished | 标准发布 | dictionary、automation |
| LineageConfirmed | 血缘确认 | impact、governance |
| RuleViolationDetected | 规则命中问题 | task、notification |
| GovernanceTaskCreated | 治理任务生成 | notification |
| ImportBatchCompleted | 导入批次完成 | automation、notification |
| CollectTaskCompleted | 采集任务完成 | diff、automation |
| ImpactAnalysisCompleted | 影响分析完成 | governance、notification |

## 5. RocketMQ / Kafka 适配

选择策略：

| 场景 | 推荐 |
| --- | --- |
| 企业内部治理事件 | RocketMQ |
| 数据平台元数据变更流 | Kafka |
| MVP 本地运行 | 本地事件总线 |

适配方式：

```text
application/domain -> EventPublisher port -> infrastructure event adapter
```

禁止业务代码直接依赖 RocketMQ 或 Kafka 客户端。

## 6. Topic 规划

建议逻辑 topic：

| Topic | 事件 |
| --- | --- |
| edd.asset.events | AssetPublished、AssetChanged |
| edd.glossary.events | BusinessTermPublished、BusinessTermDeprecated |
| edd.standard.events | StandardPublished |
| edd.lineage.events | LineageConfirmed |
| edd.governance.events | GovernanceTaskCreated、ImpactAnalysisCompleted |
| edd.collector.events | ImportBatchCompleted、CollectTaskCompleted |

## 7. 幂等与失败处理

消费者必须幂等。

幂等键：

```text
eventId + consumerName
```

失败处理：

- 短暂失败自动重试。
- 多次失败进入失败事件表。
- 支持人工重放。
- 不允许无限重试阻塞主流程。

## 8. Outbox 预留

为保证本地事务和事件发布一致性，后续可引入 Outbox 模式。

首期建议预留事件表：

- event_outbox。
- event_consume_log。
- event_failure_log。

首期本地事件可同步或异步执行，但关键事件应落库以便追溯。
