# 首期测试验收方案 v0.1

## 1. 验收目标

首期验收以治理闭环为核心，验证平台能支撑资产目录、数据字典、元数据导入、血缘影响分析、架构评审、责任认领和权限审计。

## 2. 功能验收范围

| 模块 | 验收点 |
| --- | --- |
| 资产目录 | 资产搜索、筛选、详情、标签、责任人展示 |
| 数据字典 | 字段定义维护、标准映射、状态管理、批量导入 |
| 模型管理 | 概念、逻辑、物理对象及映射关系 |
| 标准术语 | 标准创建、发布、引用、未引用清单 |
| 血缘管理 | 表级血缘、关键字段血缘、人工确认、路径展示 |
| 影响分析 | 从表、字段、接口、报表发起影响分析 |
| 流程审批 | 架构评审、模型变更、标准发布、血缘确认 |
| 权限审计 | 角色权限、数据范围、敏感字段、操作日志 |

## 3. 核心验收场景

### 3.1 系统新建架构评审

输入：

- 系统信息。
- 核心数据对象。
- 接口清单。
- 责任人。
- 分类分级。

预期：

- 平台生成评审单。
- 检查标准引用、模型复用和影响范围。
- 审批记录可追溯。

### 3.2 模型变更影响分析

输入：

- 修改核心表字段。

预期：

- 生成字段差异。
- 展示受影响接口、ETL、报表、指标和责任人。
- 影响分析结果进入审批单。

### 3.3 报表溯源

输入：

- 选择核心报表。

预期：

- 追溯到数据集、ETL、表和关键字段。
- 展示口径、责任人和更新时间。

### 3.4 责任认领

输入：

- 无 Owner 核心表。

预期：

- 生成认责任务。
- Owner 和 Steward 确认后进入已确认状态。

## 4. 非功能验收

| 类型 | 验收标准 |
| --- | --- |
| 性能 | 常规搜索秒级返回，血缘图按需加载 |
| 安全 | 未授权用户无法查看敏感字段详情 |
| 审计 | 发布、审批、导出、权限变更有日志 |
| 可用性 | 采集失败不影响资产查询 |
| 可维护性 | 新增资产类型不需要大改表结构 |

## 5. 验收证据

- 测试用例执行记录。
- 截图或录屏。
- 审批单记录。
- 资产导入记录。
- 血缘影响分析报告。
- 权限审计日志。
- 问题整改清单。
