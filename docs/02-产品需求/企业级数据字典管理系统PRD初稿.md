## 1. 文档信息

| 项目 | 内容 |
| --- | --- |

## 2. 产品背景

企业存在多系统异构、数据平台偏报表、架构资产薄弱、血缘不清、数据标准难执行、数据责任难追溯等问题。现有工具和文档无法支撑企业级数据架构治理，也难以在系统建设和变更过程中提前发现数据标准、模型复用、接口依赖和报表影响问题。

本产品旨在建立企业统一的数据字典、数据模型、元数据、数据血缘和架构治理平台，将数据治理要求嵌入系统建设和变更流程，提升企业数据资产的可见性、可控性和可运营性。

## 3. 产品目标

### 3.1 业务目标

- 支撑企业级数据架构治理和架构评审。
- 建立统一的数据字典和元数据资产目录。
- 建立概念模型、逻辑模型、物理模型的一体化管理体系。
- 支撑数据血缘追踪和变更影响分析。
- 建立数据标准、业务术语、指标口径和数据模型之间的映射。
- 建立数据资产责任体系和治理运营看板。

### 3.2 首期目标

- 完成平台基础能力建设。
- 支持核心资产对象的录入、采集、查询、维护、审批和发布。
- 支持系统、接口、表、字段、ETL、报表和指标的统一管理。
- 支持表级血缘和关键字段级血缘展示。
- 支持系统建设、模型变更、接口变更和报表指标新增的架构评审闭环。
- 支持按业务域分批纳管。

### 3.3 非目标

首期不实现以下能力：

- 不追求全企业字段级血缘一次性全覆盖。
- 不替代数据库设计工具、ETL 工具、BI 工具或数据质量平台。
- 不实现复杂主数据分发和数据交换执行能力。
- 不作为业务系统的数据录入、交易处理或报表计算平台。
- 不直接承载生产数据明细查询。

## 5. 核心业务场景

### 5.1 系统新建架构评审

业务方或项目团队新建系统时，需要在平台登记系统、业务域、数据对象、接口、核心模型、数据标准引用和责任人。企业架构师和数据架构师基于平台进行架构评审，确认是否存在重复建设、标准偏离、主数据冲突、接口影响和安全风险。

### 5.2 系统改造和数据模型变更

系统团队提交表结构、字段定义、接口字段或数据模型变更。平台自动识别相关下游接口、ETL、报表、指标和负责人，形成影响分析报告。相关 Owner、Steward、架构师完成审批后，变更才能发布。

### 5.3 报表和指标口径查询

业务用户搜索报表、指标或字段，查看业务定义、统计口径、来源表、关键字段、加工链路、责任人和更新时间，减少口径解释依赖人工沟通。

### 5.4 数据标准发布与执行

数据治理团队维护数据标准和业务术语，发布后字段、逻辑属性、指标和接口字段可以引用标准。平台对未引用标准、命名不合规或定义缺失的对象进行提示。

### 5.5 血缘影响分析

当某个字段、表、接口、ETL 任务或报表发生变更或故障时，用户可以查看上下游路径、受影响资产、影响等级、责任人和处理建议。

### 5.6 数据资产责任认领

平台根据系统、业务域、组织和资产类型生成待认领清单。相关 Owner 和 Steward 确认责任关系后，平台将责任信息用于审批、通知、影响分析和治理看板。

## 6. 功能范围

### 6.1 首页与工作台

#### 功能说明

为不同角色提供待办、资产概览、治理指标和快捷入口。

#### 主要需求

- 展示个人待办：评审、审批、认责、元数据修正、血缘确认。
- 展示资产总览：系统数、表数、字段数、接口数、报表数、指标数。
- 展示治理指标：责任认领率、标准映射率、元数据完整率、血缘覆盖率。
- 支持按角色展示不同工作台内容。
- 支持跳转到资产详情、审批单、血缘图和治理看板。

### 6.2 数据资产目录

#### 功能说明

统一展示企业数据相关资产，支持搜索、筛选、分类、标签和详情查看。

#### 主要需求

- 支持按业务域、系统、资产类型、责任人、数据分类、状态筛选。
- 支持全文搜索字段、表、系统、报表、指标、术语和标准。
- 支持资产详情页，展示基础信息、业务定义、技术属性、责任信息、血缘关系、变更记录。
- 支持资产收藏、关注和订阅变更。
- 支持资产标签，包括核心资产、敏感资产、主数据、参考数据、待治理资产。

### 6.3 数据字典管理

#### 功能说明

管理表、字段、接口字段、文件字段、报表字段和指标字段的业务定义与技术属性。

#### 主要需求

- 支持自动采集字段技术属性，包括名称、类型、长度、精度、是否主键、是否必填、默认值。
- 支持人工维护中文名、业务定义、业务规则、取值范围、枚举说明。
- 支持维护数据分类分级、敏感属性、脱敏要求。
- 支持维护责任人、所属业务域、所属系统、标准映射。
- 支持字段状态管理：待确认、已确认、已发布、待修订、废弃。
- 支持批量导入、批量导出、批量修改和变更记录。
- 支持字段差异对比，展示采集结构与已发布字典之间的差异。

### 6.4 概念模型管理

#### 功能说明

管理业务概念和高层实体关系，支撑业务架构和数据架构衔接。

#### 主要需求

- 支持创建业务域和主题域。
- 支持创建概念实体，维护名称、定义、别名、责任部门、相关业务流程。
- 支持维护概念实体之间的关系。
- 支持概念模型图形化展示。
- 支持概念实体与业务术语、逻辑实体、数据标准建立映射。
- 支持模型版本和发布状态管理。

### 6.5 逻辑模型管理

#### 功能说明

管理逻辑实体、逻辑属性和实体关系，作为业务语义到物理实现的桥梁。

#### 主要需求

- 支持创建逻辑实体和逻辑属性。
- 支持定义主键、唯一性、必填性、取值规则和业务规则。
- 支持逻辑实体之间的关系建模。
- 支持逻辑属性引用数据标准和业务术语。
- 支持逻辑模型与概念模型、物理模型之间的映射。
- 支持逻辑模型评审、版本对比和发布。

### 6.6 物理模型管理

#### 功能说明

管理数据库物理表、字段、索引、约束和存储信息。

#### 主要需求

- 支持从数据库自动采集物理模型。
- 支持展示数据库、Schema、表、字段、索引、约束、分区信息。
- 支持物理模型与逻辑模型映射。
- 支持采集结果与已发布模型对比。
- 支持物理变更提交审批。
- 支持物理模型版本记录和差异分析。

### 6.7 数据标准与业务术语管理

#### 功能说明

维护企业统一数据标准、业务术语和指标口径基础定义。

#### 主要需求

- 支持数据标准分类管理，包括基础标准、命名标准、编码标准、指标标准、主数据标准、参考数据标准。
- 支持业务术语维护，包括中文名、英文名、缩写、同义词、定义、适用范围。
- 支持标准和术语的新增、修订、发布、废止流程。
- 支持字段、逻辑属性、指标、接口字段引用标准。
- 支持标准引用统计和未引用资产清单。
- 支持命名、类型、长度、枚举和定义完整性检查。

### 6.8 系统与接口资产管理

#### 功能说明

管理企业系统、应用、模块、接口、API、消息和文件交换。

#### 主要需求

- 支持系统基础信息维护：系统名称、编码、业务域、负责人、技术负责人、生命周期状态。
- 支持系统与数据库、接口、报表、ETL、数据集之间的关系维护。
- 支持接口基础信息维护：接口名称、提供方、消费方、协议、频率、数据对象、字段清单。
- 支持 API、消息 Topic、文件交换等多类集成资产。
- 支持接口变更影响分析。
- 支持系统间数据流图展示。

### 6.9 报表与指标资产管理

#### 功能说明

管理报表、看板、数据集、指标和口径定义。

#### 主要需求

- 支持报表基础信息维护：名称、所属系统、使用部门、负责人、刷新频率、生命周期状态。
- 支持指标定义维护：名称、业务定义、计算口径、统计周期、维度、度量、责任人。
- 支持报表与指标、数据集、ETL、表、字段之间建立关系。
- 支持报表血缘追溯。
- 支持指标口径变更审批。
- 支持报表和指标影响分析。

### 6.10 元数据自动采集

#### 功能说明

通过采集任务获取数据库、ETL、BI、接口和调度工具中的技术元数据。

#### 主要需求

- 支持配置数据源连接信息。
- 支持周期性采集数据库元数据。
- 支持采集 ETL 作业、调度任务、数据集和报表元数据。
- 支持采集接口清单或通过 API 导入接口元数据。
- 支持采集任务执行记录、失败重试和异常告警。
- 支持采集结果差异识别。
- 支持将采集结果提交人工确认后发布。

### 6.11 数据血缘管理

#### 功能说明

管理和展示数据从源系统到报表指标的加工、同步、引用和派生关系。

#### 主要需求

- 支持系统级、接口级、表级、报表级和关键字段级血缘。
- 支持自动解析生成技术血缘。
- 支持人工维护和修正血缘关系。
- 支持血缘关系确认流程。
- 支持血缘图按层级展开、折叠、过滤和路径高亮。
- 支持查看上游来源、下游去向、加工任务和责任人。
- 支持血缘版本记录。

### 6.12 变更影响分析

#### 功能说明

基于元数据关系和血缘关系分析变更或故障影响范围。

#### 主要需求

- 支持从系统、接口、表、字段、ETL、报表、指标发起影响分析。
- 支持展示受影响资产、影响路径、影响等级和责任人。
- 支持生成影响分析报告。
- 支持将影响分析结果作为审批附件。
- 支持按业务域、系统、资产类型过滤影响范围。
- 支持故障场景下快速定位下游影响。

### 6.13 架构评审与审批流程

#### 功能说明

将数据治理要求嵌入系统建设和变更流程。

#### 主要需求

- 支持系统新建架构评审申请。
- 支持系统改造架构评审申请。
- 支持模型新增和变更审批。
- 支持接口新增和变更审批。
- 支持报表和指标新增审批。
- 支持数据标准发布审批。
- 支持审批节点配置、审批意见、附件、驳回、撤回、转办。
- 支持审批记录与资产版本关联。
- 支持架构例外申请和豁免记录。

### 6.14 责任人与组织管理

#### 功能说明

管理组织、用户、角色和资产责任关系。

#### 主要需求

- 支持组织和用户同步或导入。
- 支持角色配置，包括企业架构师、数据架构师、Owner、Steward、系统管理员等。
- 支持资产责任人维护。
- 支持责任认领流程。
- 支持责任变更记录。
- 支持按责任人统计待治理资产和影响资产。

### 6.15 权限与安全分级

#### 功能说明

控制不同用户对资产、定义、敏感信息和流程的访问权限。

#### 主要需求

- 支持统一认证集成。
- 支持角色权限和数据范围权限。
- 支持按业务域、系统、资产类型授权。
- 支持敏感字段展示控制。
- 支持操作审计日志。
- 支持审批、发布、删除、导出等高风险操作权限控制。

### 6.16 质量检查与运营看板

#### 功能说明

持续监控元数据、标准、责任和血缘的治理质量。

#### 主要需求

- 支持元数据完整性检查。
- 支持标准映射率统计。
- 支持责任认领率统计。
- 支持血缘覆盖率统计。
- 支持待确认、待补录、待修正资产清单。
- 支持按业务域、系统、责任人查看治理指标。
- 支持导出治理运营报告。

### 6.17 系统集成与开放 API

#### 功能说明

为外部系统提供资产查询、元数据同步、审批集成和治理数据服务。

#### 主要需求

- 支持资产查询 API。
- 支持元数据导入 API。
- 支持审批流程集成 API。
- 支持血缘查询 API。
- 支持标准和术语查询 API。
- 支持与 CMDB、IAM、DevOps、数据平台、BI、数据质量平台集成。
- 支持 API 调用鉴权和审计。

## 8. 核心流程

### 8.1 元数据采集流程

1. 管理员配置采集连接和采集范围。
2. 系统执行采集任务，获取技术元数据。
3. 平台生成新增、变更、删除差异。
4. 系统 Owner 或 Steward 确认采集结果。
5. 已确认结果发布到资产目录和数据字典。
6. 平台记录采集批次、确认人、发布时间和差异详情。

### 8.2 数据标准发布流程

1. 数据治理人员创建或修订数据标准。
2. 相关 Owner、架构师和治理负责人评审。
3. 审批通过后标准发布。
4. 平台推送受影响资产和待映射清单。
5. Steward 完成标准映射和整改。

### 8.3 模型变更审批流程

1. 系统团队或数据架构师提交模型变更。
2. 平台自动生成差异和影响分析。
3. 数据架构师评审模型合理性。
4. 数据 Owner 确认业务影响。
5. 企业架构师确认架构合规。
6. 审批通过后发布新版本。

### 8.4 系统建设架构评审流程

1. 项目团队登记系统建设信息。
2. 平台要求填写业务域、数据对象、接口、模型、责任人和安全分级。
3. 平台检查是否引用数据标准、是否存在重复模型、是否存在下游影响。
4. 企业架构师、数据架构师、安全合规人员评审。
5. 评审通过后形成架构评审记录。

### 8.5 血缘确认流程

1. 平台通过采集和解析生成初始血缘。
2. 相关系统 Owner、数据平台负责人或 Steward 查看待确认血缘。
3. 用户修正或补充血缘关系。
4. 血缘状态变为已确认。
5. 已确认血缘用于影响分析和报表溯源。

### 8.6 数据资产认责流程

1. 平台生成无责任人或责任异常资产清单。
2. 按系统、业务域或组织分配认责任务。
3. 责任人确认 Owner、Steward 和维护人。
4. 平台更新责任关系并记录变更。
5. 责任信息用于审批、通知、看板和影响分析。

## 10. 权限设计

### 10.1 权限类型

- 菜单权限：控制功能入口。
- 操作权限：控制新增、编辑、删除、发布、审批、导出。
- 数据权限：按业务域、系统、组织、资产类型控制数据范围。
- 敏感权限：控制敏感字段、分类分级和安全属性展示。
- 流程权限：控制审批节点、转办、驳回、豁免和终止。

### 10.2 默认角色

| 角色 | 权限范围 |
| --- | --- |
| 平台管理员 | 系统配置、用户角色、数据源、采集任务、全局权限 |
| 企业架构师 | 架构视图、架构评审、系统依赖、合规检查 |
| 数据架构师 | 模型管理、模型评审、数据域维护、模型发布 |
| 数据治理管理员 | 标准术语、责任体系、质量检查、治理看板 |
| 数据 Steward | 字典维护、标准映射、血缘确认、责任维护 |
| 系统 Owner | 本系统资产维护、接口确认、变更提交 |
| 数据 Owner | 业务定义确认、变更审批、责任确认 |
| 普通用户 | 已授权资产查询、定义查看、血缘查看 |

## 11. 数据状态与版本

### 11.1 资产状态

- 待采集。
- 待确认。
- 已确认。
- 已发布。
- 待修订。
- 已废弃。

### 11.2 流程状态

- 草稿。
- 审批中。
- 已通过。
- 已驳回。
- 已撤回。
- 已关闭。

### 11.3 版本要求

- 数据标准、业务术语、模型、字典和血缘关系需要记录版本。
- 每次发布应记录发布人、发布时间、变更说明和关联审批单。
- 支持查看历史版本和版本差异。
- 采集元数据与治理发布版本需要区分展示。

## 12. 非功能需求

### 12.1 性能

- 资产搜索应支持秒级响应。
- 常规资产详情页应在可接受时间内加载完成。
- 血缘图需要支持按需展开，避免一次性加载全图导致不可用。
- 采集任务应支持异步执行和后台监控。

### 12.2 可扩展性

- 元模型应支持扩展资产类型、属性和关系。
- 采集器应支持插件化扩展。
- 流程节点和审批规则应支持配置。
- 权限模型应支持组织调整和业务域扩展。

### 12.3 可用性

- 采集失败不应影响平台核心查询能力。
- 血缘解析失败应可人工补录和修正。
- 审批流程异常应支持管理员干预。
- 关键操作需要有明确提示和回滚记录。

### 12.4 安全

- 支持统一认证和单点登录集成。
- 支持最小权限原则。
- 敏感字段默认受控展示。
- 导出、删除、发布、审批等操作必须记录审计日志。
- API 调用必须鉴权并记录调用日志。

### 12.5 兼容性

- 支持主流关系型数据库元数据采集。
- 支持通过文件或 API 导入非标准系统元数据。
- 支持与企业现有 IAM、CMDB、DevOps、数据平台、BI 平台集成。

## 13. 首期验收标准

### 13.1 功能验收

- 能创建和维护系统、数据库、表、字段、接口、ETL、报表、指标等资产。
- 能创建和维护概念模型、逻辑模型、物理模型及映射关系。
- 能创建、发布和引用数据标准与业务术语。
- 能配置并执行元数据采集任务。
- 能展示表级血缘和关键字段级血缘。
- 能发起架构评审、模型变更审批和标准发布审批。
- 能执行变更影响分析并生成影响清单。

### 13.2 治理闭环验收

- 系统新建场景可完成架构评审闭环。
- 模型变更场景可完成影响分析、审批和发布闭环。
- 数据标准发布后可跟踪引用和整改情况。
- 数据资产可完成 Owner 和 Steward 认责。
- 关键报表可追溯到上游表和关键字段。

### 13.3 数据质量验收

- 首批业务域核心资产具备中文名、业务定义、技术属性、责任人和更新时间。
- 首批核心字段具备标准映射或待整改标记。
- 首批核心报表具备来源关系和负责人。
- 首批核心系统具备接口和数据流关系。

### 13.4 使用验收

- 企业架构师可以完成架构评审。
- 数据架构师可以完成模型管理和模型评审。
- 数据治理人员可以维护标准、术语、责任和质量指标。
- 系统团队可以提交和跟踪数据模型变更。
- 普通用户可以查询字段、指标、报表口径和来源。
