# 产品功能全景与自动化业务说明 v0.1

## 1. 产品定位

企业级数据字典管理系统不是单一的数据字典 CRUD 工具，而是面向企业数据架构治理、元数据管理、数据血缘、架构评审和数据资产运营的综合平台。

平台核心价值：

- 以资产目录统一呈现企业数据资产。
- 以数据字典承载字段定义、技术属性、业务语义和责任信息。
- 以元模型支撑资产类型、属性、关系、状态和版本扩展。
- 以治理流程推动架构评审、模型变更、标准发布、血缘确认和责任认领。
- 以自动化引擎支撑采集、差异识别、规则校验、任务生成、通知和整改闭环。

## 3. 必要功能

首期 MVP 必须具备：

- 资产目录。
- 数据字典。
- 元模型管理。
- Excel 导入。
- 数据库元数据采集框架。
- 业务术语。
- 数据标准。
- 责任体系。
- 基础血缘。
- 影响分析。
- 审批流程。
- 权限审计。
- 状态和版本管理。

没有这些能力，平台无法形成架构治理闭环。

## 4. 高难度功能

| 功能 | 难点 |
| --- | --- |
| 统一元模型 | 抽象和落地之间需要平衡，不能过度抽象也不能写死 |
| 采集与人工治理融合 | 技术事实和治理定义冲突时必须有差异、确认、发布机制 |
| 血缘可信度 | 自动解析、人工维护、导入来源可信度不同，必须可确认和分级 |
| 影响分析 | 不只是图遍历，还要识别影响等级、责任人、核心资产和阻断风险 |
| 三层模型映射 | 概念、逻辑、物理模型之间要可追溯，不能停留在静态文档 |
| 标准执行闭环 | 标准发布后必须找到未映射、不合规、需整改资产 |
| 业务术语治理 | 术语需要统一业务语言，并处理同名不同义、同义不同名、缩写冲突和术语影响 |
| 权限叠加 | 菜单、操作、数据范围、敏感字段权限需要集中设计 |
| 审批与状态机 | 审批通过后必须驱动资产、模型、标准、血缘状态和版本变化 |
| Excel 导入 | 模板版本、行级错误、重复编码、引用校验、增量更新都复杂 |
| 自动化治理 | 自动识别、自动校验、自动任务和自动通知必须可控、可解释 |

## 5. 平台自动化业务定义

平台自动化业务不是单纯自动采集，而是完整治理闭环：

```text
发现变化
-> 识别差异
-> 执行规则
-> 生成任务
-> 推动确认
-> 进入审批
-> 整改闭环
-> 持续监控
```

自动化分为 8 类：

| 自动化类别 | 说明 |
| --- | --- |
| 自动采集 | 从数据库、ETL、BI、接口、CMDB 等系统获取元数据 |
| 自动识别 | 识别新增、修改、删除、冲突、缺失、敏感字段、核心资产 |
| 自动校验 | 校验命名规范、必填信息、标准引用、责任归属、分类分级 |
| 自动比对 | 比对采集现状、导入数据和平台已发布治理定义 |
| 自动分析 | 基于血缘和资产关系执行影响分析、风险判断 |
| 自动生成 | 生成待确认任务、整改任务、审批检查项、影响报告 |
| 自动流转 | 根据规则触发审批、会签、认责、血缘确认、例外申请 |
| 自动通知 | 向 Owner、Steward、架构师、安全人员推送待办、风险和变更 |

## 6. 已纳入平台的自动化业务

| 自动化业务 | 说明 |
| --- | --- |
| 元数据自动采集 | 采集数据库、Schema、表、字段、索引、约束、注释 |
| 导入批次自动校验 | 校验字段完整性、枚举值、重复编码、引用对象 |
| 采集差异自动识别 | 识别新增、修改、删除、冲突、缺失 |
| 资产状态自动流转 | 导入或采集后进入待确认，确认后发布 |
| 标准引用自动检查 | 检查核心字段、逻辑属性、指标、接口字段是否引用标准 |
| 术语引用自动推荐 | 根据字段名、指标名、定义、业务域推荐可引用业务术语 |
| 术语冲突自动识别 | 识别同名不同义、同义不同名、缩写冲突、跨域定义冲突 |
| 术语影响自动分析 | 术语修订或废止前自动识别受影响字段、指标、报表、模型 |
| 元数据完整性自动检查 | 检查中文名、业务定义、责任人、标准映射是否缺失 |
| 敏感字段自动识别建议 | 基于字段名、定义、类型和规则生成敏感建议 |
| 分类分级自动建议 | 根据敏感识别和规则建议安全分级 |
| 血缘自动生成 | 通过采集、SQL、ETL 配置、BI 元数据、接口导入生成血缘 |
| 血缘确认任务自动生成 | 自动生成待确认血缘任务 |
| 影响分析自动生成 | 从表、字段、接口、ETL、报表、指标发起影响分析 |
| 审批检查项自动生成 | 架构评审或模型变更时生成检查项 |
| 整改任务自动生成 | 对标准缺失、定义缺失、责任缺失、血缘断点生成整改任务 |
| 责任认领任务自动生成 | 对无 Owner 或 Steward 的资产生成认责任务 |
| 治理指标自动统计 | 统计完整率、映射率、认领率、血缘覆盖率 |
| 订阅变更自动通知 | 关注资产变更时自动通知订阅人 |
| 采集任务自动调度 | 周期性执行采集任务并记录日志 |
| 审计日志自动记录 | 自动记录导入、导出、发布、审批、敏感字段查看、权限变更 |

## 7. 需补强的产品模块

### 7.1 采集差异工作台

用于展示采集或导入后的差异。

差异类型：

- 新增。
- 修改。
- 删除。
- 冲突。
- 缺失。

操作：

- 确认。
- 忽略。
- 修正。
- 发起审批。
- 发布。

### 7.2 问题台账与整改任务

问题来源：

- 规则校验。
- 人工发现。
- 采集差异。
- 审批驳回。
- 血缘断点。
- 安全风险。

任务字段：

- 问题类型。
- 影响资产。
- 责任人。
- 截止时间。
- 严重等级。
- 整改状态。
- 处理记录。

### 7.3 治理规则配置中心

规则类型：

- 元数据完整性。
- 标准映射。
- 命名规范。
- 敏感识别。
- 分类分级。
- 影响等级。
- 审批路由。
- 治理评分。

规则必须支持版本、启停、适用范围和执行日志。

### 7.4 通知中心

通知类型：

- 待办提醒。
- 审批通知。
- 认责通知。
- 血缘确认通知。
- 整改通知。
- 采集异常。
- 资产订阅变更。
- 例外到期提醒。

### 7.5 例外与豁免管理

适用场景：

- 暂不符合架构原则。
- 暂不引用数据标准。
- 暂不补齐血缘。
- 暂不完成分类分级。

要求：

- 必须有原因。
- 必须有责任人。
- 必须有有效期。
- 到期自动提醒。
- 可纳入治理看板。

## 8. 自动化边界

自动化可以建议、发现、生成和提醒，但不应绕过治理责任。

原则：

- 自动识别不等于自动决策。
- 自动差异不直接覆盖已发布治理定义。
- 自动任务必须可解释。
- 自动审批路由必须可追溯。
- 自动规则必须有版本。
- 自动化异常不得阻断核心查询能力。
