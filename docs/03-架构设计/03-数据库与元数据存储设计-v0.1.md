# 数据库与元数据存储设计 v0.1

## 1. 设计原则

- 核心资产采用统一资产表承载，避免每类资产都建独立主表导致扩展困难。
- 高频查询字段结构化存储，扩展属性采用属性表或 JSONB。
- 资产关系统一建边，支持血缘、映射、引用、责任、包含等关系。
- 版本、状态、审计独立设计，保证发布对象可追溯。
- 首期优先 PostgreSQL，后续按规模引入图数据库。

## 2. 核心表分组

| 分组 | 表 |
| --- | --- |
| 元模型 | meta_type、meta_attribute、meta_relation_type |
| 资产实例 | asset、asset_attribute_value、asset_version |
| 关系实例 | asset_relation、lineage_relation |
| 标准术语 | data_standard、business_term、standard_reference |
| 流程治理 | workflow_instance、approval_task、review_check_item |
| 权限审计 | sys_user、sys_role、sys_permission、sys_audit_log |
| 采集导入 | import_batch、import_error、collect_task、collect_snapshot |

## 7. 索引建议

- asset：asset_type、code、name、display_name、business_domain、system_code、status。
- asset_relation：from_asset_id、to_asset_id、relation_type。
- lineage_relation：from_asset_id、to_asset_id、lineage_level、confirm_status。
- asset_attribute_value：asset_id、attr_code。
- sys_audit_log：operator_id、object_type、object_id、created_at。

## 8. 删除策略

- 核心资产不物理删除，采用状态废弃。
- 导入批次和错误明细可按归档策略清理。
- 审批、审计、发布版本不得删除。
