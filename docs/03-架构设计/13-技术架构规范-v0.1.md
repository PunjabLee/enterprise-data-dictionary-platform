# 技术架构规范 v0.1

## 1. 定位

本文档定义平台首期技术架构规范，用于约束技术栈、组件依赖、工程结构、端口适配、部署运行、安全审计、非功能和可观测性。

本规范不改变既有 ADR：首期采用 DDD 分层的模块化单体、PostgreSQL 主存储、本地领域事件和可插拔 MQ、搜索、工作流适配。

## 2. 技术栈冻结

| 层次 | 技术 | 规范 |
| --- | --- | --- |
| 后端语言 | Java 21 | 统一使用 Java 21，不引入多语言后端 |
| 后端框架 | Spring Boot 3 | 一个模块化单体部署单元 |
| 构建 | Maven | 不使用 Gradle 作为首期构建系统 |
| 数据访问 | MyBatis-Plus | Mapper 属于 infrastructure，不进入 domain |
| 数据库迁移 | Flyway | 所有 DDL 通过 Flyway 管理 |
| 主库 | PostgreSQL 16 | 资产、元模型、关系、流程、权限、审计事实源 |
| 缓存 | Redis | 缓存、权限快照、短期任务状态，不作为事实源 |
| 前端 | Vue 3 + TypeScript + Vite + Ant Design Vue | 企业控制台前端技术基线 |
| 导入导出 | Apache POI | Excel 模板、导入校验、导出 |
| 调度 | Spring Scheduler / Quartz | 首期用于采集任务、异步任务调度 |

## 3. 组件依赖分级

| 组件 | 分级 | 规范 |
| --- | --- | --- |
| PostgreSQL | 强依赖 | 生产事实源，必须可备份、可恢复、可迁移 |
| Redis | 强依赖 | 用于缓存和辅助状态，任何数据必须可从 PostgreSQL 重建 |
| Flyway | 强依赖 | 所有环境统一执行迁移 |
| 本地事件表/事件总线 | 强依赖 | 首期事件闭环基础 |
| OpenSearch | 预留适配 | 首期不强依赖，搜索先基于 PostgreSQL 索引，保留 SearchGateway |
| Flowable | 预留适配 | 首期轻量审批任务，复杂流程后续适配 |
| RocketMQ/Kafka | 可选适配 | 通过 DomainEventPublisher/Subscriber 端口接入 |
| 图数据库 | 后续演进 | 首期血缘使用 PostgreSQL 关系表 |
| 对象存储 | 可选适配 | 首期可用本地文件，后续适配 S3/MinIO |

## 4. 工程结构规范

后端工程应采用 Maven 多模块或清晰包模块组织，最小结构：

```text
platform/backend/
  pom.xml
  metadata-platform/
    src/main/java/
    src/main/resources/db/migration/
    src/test/java/
```

包结构必须对齐应用架构：

```text
com.<org>.dictionary.<context>.<layer>
```

约束：

- `domain` 不依赖 Spring、MyBatis、Redis、MQ、HTTP 客户端。
- `interfaces` 不直接访问 Mapper。
- `infrastructure` 实现端口，不向 domain 泄露基础设施类型。
- 公共类型优先放在明确的 common 包，禁止把业务规则放入 common。
- 架构测试应使用 ArchUnit 或等价机制验证依赖方向。

## 5. 可插拔端口规范

| 能力 | 端口 | 首期实现 | 后续适配 |
| --- | --- | --- | --- |
| 领域事件 | DomainEventPublisher、DomainEventSubscriber | LocalDomainEventPublisher | RocketMQ、Kafka |
| 搜索 | SearchGateway、AssetSearchIndexer | PostgreSQL 查询/索引 | OpenSearch |
| 工作流 | WorkflowEngine | LightweightWorkflowEngine | Flowable |
| 规则 | RuleEngine、RuleExecutor | 内置规则执行器 | Drools 或脚本引擎 |
| 文件 | FileStoragePort | LocalFileStorage | MinIO、S3 |
| 采集 | CollectorAdapter | Excel/API/基础 JDBC 适配 | ETL、BI、调度平台适配 |
| 通知 | NotificationSender | 站内通知 | 邮件、IM、Webhook |

端口约束：

- 业务代码只能依赖端口，不直接依赖 MQ/Search/Workflow/File SDK。
- 端口入参和出参使用平台内部模型或 DTO，不泄露三方 SDK 类型。
- 可选组件不可影响首期本地启动。
- 适配器失败必须可记录、可重试、可降级。

## 6. 数据与事务规范

- PostgreSQL 是唯一强事实源。
- Redis 不保存不可重建的业务事实。
- 写操作由 application service 定义事务边界。
- 跨上下文一致性优先采用本地事务 + 领域事件 + 幂等消费者。
- 已发布资产、模型、标准、术语、血缘不得原地覆盖，应通过版本和状态管理。
- Flyway 脚本必须可重复在 local、dev、test、prod 初始化和升级。

## 7. 安全与审计规范

| 主题 | 规范 |
| --- | --- |
| 认证 | 首期支持本地账号/JWT，预留 OIDC/LDAP/AD |
| 授权 | RBAC + 数据范围 + 敏感字段权限 |
| 密钥 | 数据源密码、JWT secret、集成密钥不得提交仓库 |
| 采集账号 | 数据库采集账号必须只读 |
| 导出控制 | L3 及以上数据导出必须授权和审计 |
| 审计 | 发布、审批、导入、导出、删除、权限变更、敏感分级变更必须审计 |
| 文件 | 导入文件、导出文件、敏感附件必须有访问控制和清理策略 |

## 8. 部署运行规范

环境分层：

| 环境 | 用途 | 规范 |
| --- | --- | --- |
| local | 本地开发 | Docker Compose 起步，允许使用本地文件存储 |
| dev | 联调 | 使用独立 PostgreSQL/Redis，保留样例数据 |
| test | 测试验收 | 启用 Flyway、审计、权限和导入导出测试 |
| prod | 生产 | 外部化配置、备份恢复、监控告警、禁止默认密码 |

部署约束：

- 本地 Docker Compose 至少包含 PostgreSQL、Redis、后端、前端。
- OpenSearch、MQ、Flowable 可作为可选 profile。
- 敏感配置通过环境变量或密钥管理注入。
- 生产必须开启审计日志和备份策略。

## 9. 可观测性与非功能规范

| 类型 | 目标 |
| --- | --- |
| 性能 | 常用资产检索秒级返回；万级 Excel 导入采用分批和异步任务 |
| 可用性 | 异步任务失败可重试，导入批次可追踪 |
| 可扩展 | 新资产类型优先通过元模型配置扩展 |
| 可观测 | API、任务、事件、审计日志必须贯穿 traceId |
| 日志 | 应用日志、访问日志、任务日志、采集日志、审计日志分层记录 |
| 指标 | API 错误率、DB 连接池、任务失败率、导入耗时、磁盘空间、事件失败数 |
| 备份 | PostgreSQL 每日备份，关键阶段手工快照 |

## 10. 禁止事项

- 禁止首期引入微服务拆分作为默认实现。
- 禁止业务代码直接依赖 RocketMQ/Kafka/OpenSearch/Flowable 客户端。
- 禁止绕过 Flyway 手工维护生产 DDL。
- 禁止将 Redis 作为事实源。
- 禁止把敏感配置提交到仓库。
- 禁止采集账号拥有写权限。
- 禁止管理员绕过分支策略将 owner/docs/feature 分支合入 main。
