## 5. 首期模块优先级

| 优先级 | 模块 | 说明 |
| --- | --- | --- |
| P0 | 用户、角色、权限、审计 | 平台基础能力 |
| P0 | 元模型、资产、关系 | 所有业务能力底座 |
| P0 | 资产目录、数据字典 | 用户可见的核心价值 |
| P0 | Excel 导入导出 | 等待自动采集前的低成本入口 |
| P1 | 数据库采集 | 自动获取物理元数据 |
| P1 | 标准术语、责任认领 | 治理闭环 |
| P1 | 血缘关系、影响分析 | 架构治理关键能力 |
| P2 | 工作流、评审审批 | 强化流程闭环 |
| P2 | 搜索索引、治理看板 | 提升使用体验 |

## 6. 编码约定

- 后端所有接口返回统一响应结构，包含 `code`、`message`、`data`、`traceId`。
- 所有业务对象保留 `createdBy`、`createdAt`、`updatedBy`、`updatedAt`、`status`。
- 关键对象保留版本号，发布类对象不得直接物理删除。
- 导入、采集、审批、发布、权限变更必须记录审计日志。
- 前端类型定义与后端 DTO 对齐，避免页面直接拼接松散对象。

## 7. 环境划分

| 环境 | 用途 |
| --- | --- |
| local | 开发本地环境，使用 Mock 数据和 Docker Compose |
| dev | 联调环境，接入测试数据库和测试 IAM |
| test | 测试验收环境，执行完整流程和权限验证 |
| prod | 生产环境，接入真实元数据采集和正式权限 |

## 8. 代码启动条件

正式编码前需要确认：

- 是否接受推荐技术基线：Java 21、Spring Boot 3、Vue 3、PostgreSQL、OpenSearch。
- 企业是否有强制技术栈、容器平台、数据库或安全规范。
- 是否允许首期采用模块化单体。
- 是否需要接入现有统一登录和流程平台。
