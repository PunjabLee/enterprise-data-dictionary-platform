# 技术架构设计 v0.1

## 1. 设计目标

平台技术架构需要支撑企业级数据字典、元数据管理、数据模型管理、数据血缘、架构治理流程和数据资产运营。首期设计重点不是追求一次性覆盖所有系统，而是建立可扩展、可集成、可运营的平台底座。

## 2. 架构原则

- 元模型驱动：资产类型、属性、关系、状态和校验规则尽量配置化。
- 插件化采集：数据库、ETL、BI、接口等采集能力通过适配器扩展。
- 关系优先：资产、模型、标准、责任、血缘和审批记录必须可追溯。
- 流程嵌入：系统建设、模型变更、标准发布、血缘确认必须形成闭环。
- 权限前置：敏感字段、导出、审批、发布等能力从首期开始纳入权限控制。
- MVP 可运行：首期用最少组件跑通资产目录、数据字典、采集、血缘和评审闭环。

## 3. 推荐技术基线

| 层次 | 推荐方案 | 说明 |
| --- | --- | --- |
| 前端 | Vue 3 + TypeScript + Ant Design Vue | 企业后台成熟方案，组件丰富 |
| 后端 | Java 21 + Spring Boot 3 | 适合企业级权限、流程、集成和长期维护 |
| 主库 | PostgreSQL | 保存资产、元模型、权限、流程、版本和配置 |
| 搜索 | OpenSearch | 支撑资产、字段、术语、指标全文检索 |
| 血缘关系 | PostgreSQL 关系表 + 后续可插拔图存储 | 首期降低部署复杂度，后续可接 JanusGraph 等 |
| 工作流 | Flowable | Apache 2.0，适合审批、评审、认责流程 |
| 采集调度 | Quartz 或企业现有调度平台 | 首期内置，后续可接入统一调度 |
| 文件处理 | Apache POI | Excel 导入导出 |
| 图可视化 | AntV G6 或前端同类图组件 | 支撑血缘、模型和系统数据流展示 |
| 部署 | Docker Compose 起步，后续 Kubernetes | 先降低试点部署成本 |

## 4. 逻辑架构

| 层次 | 核心模块 |
| --- | --- |
| 门户层 | 工作台、资产搜索、资产详情、血缘图、模型图、治理看板 |
| 业务能力层 | 资产目录、数据字典、模型管理、标准术语、血缘管理、影响分析 |
| 治理流程层 | 架构评审、模型变更、标准发布、血缘确认、责任认领、例外审批 |
| 元数据服务层 | 元模型、资产实例、关系实例、版本、状态、标签、质量规则 |
| 集成采集层 | 数据库采集、Excel 导入、ETL/BI/API 适配器、开放 API |
| 基础支撑层 | 用户权限、审计日志、任务调度、通知、文件存储、配置中心 |

## 5. 核心服务划分

| 服务 | 职责 |
| --- | --- |
| metadata-service | 元模型、资产实例、属性、关系、版本管理 |
| catalog-service | 资产目录、搜索、详情聚合、标签、收藏 |
| governance-service | 流程、审批、评审、认责、发布、例外管理 |
| lineage-service | 血缘关系、路径查询、影响分析、血缘确认 |
| standard-service | 数据标准、业务术语、指标口径、引用检查 |
| collector-service | 采集任务、适配器、差异识别、采集日志 |
| security-service | 用户、角色、权限、数据范围、审计 |
| integration-api | 对外开放 API、导入导出、第三方系统集成 |

首期可采用模块化单体实现，按上述服务边界组织包结构和数据库 Schema；当规模扩大后再拆分为独立服务。

## 6. 数据存储设计

| 存储 | 存放内容 |
| --- | --- |
| PostgreSQL | 元模型、资产、属性、关系、审批、权限、版本、配置 |
| OpenSearch | 资产搜索索引、字段搜索、术语搜索、报表指标搜索 |
| 对象存储或本地文件 | 导入模板、附件、采集快照、审批附件 |
| 可选图存储 | 大规模血缘路径分析和复杂图查询 |

首期血缘采用关系表建边，字段包括 `from_asset_id`、`to_asset_id`、`relation_type`、`lineage_level`、`confidence`、`source_type`、`status`、`version`。

## 7. 集成边界

| 外部系统 | 集成方式 |
| --- | --- |
| IAM/AD/LDAP/OIDC | 用户和组织同步、单点登录 |
| CMDB | 系统、应用、负责人、生命周期同步 |
| 数据库 | 元数据采集，只读访问系统表或信息视图 |
| ETL/调度平台 | 任务、依赖、脚本位置、调度关系采集 |
| BI 平台 | 报表、数据集、指标、使用部门、刷新频率采集 |
| DevOps/上线流程 | 变更单、发布单、架构评审结果集成 |
| 数据质量平台 | 质量规则、质量结果、问题单关联 |

## 8. 安全设计

- 支持统一认证接入，首期保留本地账号兜底。
- 权限分为菜单权限、操作权限、数据范围权限和敏感字段权限。
- 导出、删除、发布、审批、权限变更必须记录审计日志。
- 敏感字段默认隐藏具体说明，只展示脱敏后的元数据摘要。
- 采集连接信息必须加密保存，禁止明文展示密码。

## 9. 非功能目标

| 能力 | 首期目标 |
| --- | --- |
| 搜索 | 常规资产搜索秒级响应 |
| 血缘图 | 默认按需展开，避免一次性加载全图 |
| 导入 | 支持万级行 Excel 分批导入 |
| 采集 | 采集任务异步执行，可查看日志和差异 |
| 扩展 | 新增资产类型和属性不需要改数据库表结构 |
| 审计 | 关键操作可追溯到人、时间、对象和变更内容 |

## 10. 首期技术风险

| 风险 | 应对 |
| --- | --- |
| 元模型过度抽象导致实现复杂 | 固定核心对象，扩展属性配置化 |
| 血缘图过大导致不可用 | 默认聚合展示，按需展开，限制层级 |
| 采集适配器范围失控 | 首期只支持数据库和 Excel 导入，ETL/BI 先做接口规范 |
| 流程过度复杂 | 首期流程只实现架构评审、模型变更、标准发布、血缘确认、认责 |
