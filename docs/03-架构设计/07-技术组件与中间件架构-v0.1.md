# 技术组件与中间件架构 v0.1

## 1. 设计目标

本文档定义平台涉及的技术组件和中间件定位，明确首期强依赖、可选组件和后续扩展组件，避免在 MVP 阶段引入过多基础设施复杂度。

## 2. 组件分级

| 组件 | 首期定位 | 说明 |
| --- | --- | --- |
| PostgreSQL 16 | 强依赖 | 元数据、资产、关系、流程、权限、审计的事实源 |
| Redis | 强依赖 | 缓存、分布式锁、任务状态、权限快照 |
| Flyway | 强依赖 | 数据库版本迁移 |
| Maven | 强依赖 | 后端依赖和构建管理 |
| MyBatis-Plus | 强依赖 | 数据访问基础能力 |
| Apache POI | 强依赖 | Excel 导入导出 |
| Quartz/Spring Scheduler | 强依赖 | 采集、扫描、规则执行等定时任务 |
| OpenSearch | 预留适配 | 首期不强依赖，先定义 SearchGateway |
| Flowable | 预留适配 | 首期轻量审批任务，后续复杂流程接入 |
| RocketMQ | 可选适配 | 企业业务事件总线优先选择 |
| Kafka | 可选适配 | 数据平台事件和元数据变更流优先选择 |
| 规则引擎 | 轻量实现 | 首期配置化规则，后续可接 Drools/QLExpress/Aviator |
| 对象存储 | 可选 | 附件、导入文件、报告文件；首期可本地文件 |

## 3. Redis 使用设计

使用场景：

- 热点字典和配置缓存。
- 用户权限快照。
- 数据范围缓存。
- 异步任务短期状态。
- 分布式锁。
- 幂等标记。

约束：

- Redis 不作为事实数据源。
- 所有 key 必须有命名规范。
- 缓存必须有 TTL 或明确失效机制。
- 权限相关缓存必须支持主动失效。

推荐 key：

```text
edd:dict:<dictType>
edd:user-permission:<userId>
edd:data-scope:<userId>
edd:task-status:<taskId>
edd:idempotent:<bizKey>
edd:lock:<resourceType>:<resourceId>
```

## 4. RocketMQ / Kafka 定位

平台不在首期同时强依赖 RocketMQ 和 Kafka。

选择原则：

| 场景 | 推荐 |
| --- | --- |
| 企业内部业务事件、审批通知、治理任务 | RocketMQ |
| 数据平台事件、采集流、元数据变更流 | Kafka |
| MVP 本地演示 | 本地事件总线 |

代码层必须依赖抽象：

```text
DomainEventPublisher
DomainEventSubscriber
```

基础设施层提供适配：

```text
LocalDomainEventPublisher
RocketMqDomainEventPublisher
KafkaDomainEventPublisher
```

## 5. OpenSearch 定位

首期不强依赖 OpenSearch。

原因：

- MVP 阶段资产规模有限，PostgreSQL 索引可以支撑基础搜索。
- 过早引入搜索集群会增加部署和同步复杂度。
- 搜索索引不是事实源，必须可重建。

必须预留：

```text
SearchGateway
AssetSearchIndexer
SearchDocument
```

后续启用条件：

- 资产数、字段数达到 PostgreSQL 搜索性能瓶颈。
- 需要高质量中文分词、模糊匹配、相关性排序。
- 需要跨资产全文检索和高亮。

## 6. Flowable 定位

首期不强依赖 Flowable 运行时。

首期采用轻量审批任务模型：

- 审批单。
- 审批节点。
- 审批任务。
- 审批意见。
- 状态机。

预留：

```text
WorkflowEngine
WorkflowDefinition
WorkflowInstance
WorkflowTask
```

后续接入 Flowable 的条件：

- 流程节点高度动态化。
- 存在复杂并行、会签、条件分支、超时升级。
- 企业已有 Flowable 或统一流程平台。

## 7. 规则引擎定位

首期实现轻量规则配置：

- 规则元数据。
- 适用范围。
- 触发时机。
- 条件表达式。
- 执行动作。
- 规则版本。
- 命中记录。

后续可接入：

- Drools。
- QLExpress。
- Aviator。
- Easy Rules。

业务代码只依赖：

```text
RuleEngine
RuleExecutionContext
RuleExecutionResult
```

## 8. 文件与对象存储

文件类型：

- 导入文件。
- 错误明细导出。
- 审批附件。
- 采集快照。
- 影响分析报告。
- 治理报告。

首期可本地文件存储，生产建议接对象存储或企业文件服务。

约束：

- 文件元数据入库。
- 文件访问受权限控制。
- 敏感文件下载必须审计。
- 临时文件要定期清理。

## 9. 调度与异步任务

使用场景：

- 元数据采集。
- 规则扫描。
- 影响分析。
- 批量导入。
- 批量导出。
- 通知重试。
- 例外到期检查。

首期可以使用 Spring Scheduler 或 Quartz。任务状态统一入库。

任务状态：

```text
pending
running
success
failed
canceled
```
