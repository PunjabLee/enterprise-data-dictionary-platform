# 部署与运行架构设计 v0.1

## 1. 部署目标

首期部署目标是支持开发、测试和生产三类环境，保证平台可运行、可监控、可备份、可追溯。部署架构应从 Docker Compose 起步，后续平滑迁移 Kubernetes。

## 2. 环境划分

| 环境 | 用途 | 数据 |
| --- | --- | --- |
| local | 开发调试 | Mock 数据、样例导入文件 |
| dev | 联调 | 测试数据库和测试用户 |
| test | 验收 | 首批试点资产脱敏数据 |
| prod | 生产 | 正式元数据、流程和审计 |

## 3. 组件部署

| 组件 | 首期部署方式 | 说明 |
| --- | --- | --- |
| Web 前端 | Nginx 容器 | 静态资源和反向代理 |
| 后端应用 | Spring Boot 容器 | 模块化单体 |
| PostgreSQL | 独立数据库或容器 | 生产建议独立数据库 |
| OpenSearch | 可选容器 | 搜索量不大时可后置 |
| Redis | 可选容器 | 缓存、会话、短期任务状态 |
| 文件存储 | 本地挂载或对象存储 | 模板、附件、导入文件 |

## 4. 网络与访问

- 用户通过浏览器访问 Web 前端。
- 前端通过 API 网关或 Nginx 访问后端。
- 后端访问 PostgreSQL、OpenSearch、Redis 和文件存储。
- 后端通过只读账号访问数据库元数据。
- 后端对外调用 IAM、CMDB、流程平台或 BI/ETL API。

## 5. 配置管理

配置分为：

- 应用配置：端口、日志、上传限制、分页大小。
- 数据库配置：连接地址、账号、连接池。
- 安全配置：JWT、SSO、密码策略、密钥。
- 采集配置：数据源、采集周期、采集范围。
- 集成配置：IAM、CMDB、BI、ETL、API 网关。

敏感配置不得提交到代码仓库，应通过环境变量、密钥管理或部署平台注入。

## 6. 日志与监控

必须记录：

- 应用日志。
- 访问日志。
- 审计日志。
- 采集任务日志。
- 导入任务日志。
- 流程审批日志。

建议监控：

- 应用存活。
- API 错误率。
- 数据库连接池。
- 导入和采集任务失败率。
- 搜索索引状态。
- 磁盘空间。

## 7. 备份恢复

| 对象 | 策略 |
| --- | --- |
| PostgreSQL | 每日全量备份，关键阶段手工快照 |
| 文件附件 | 与数据库同周期备份 |
| 配置文件 | 版本化管理，敏感配置独立备份 |
| 搜索索引 | 可通过数据库重建，低优先级备份 |

## 8. 生产上线要求

- 生产数据库初始化脚本已评审。
- 管理员账号和初始角色已配置。
- 审计日志开启。
- 导入导出权限受控。
- 采集账号为只读权限。
- 备份恢复演练完成。
- 核心接口具备健康检查。
