# 数据库物理模型 v0.2

## 1. 设计目标

本文档定义首期可用于 Flyway 迁移脚本落地的核心物理表模型。所有表默认使用 PostgreSQL 16。

## 2. 通用字段

核心业务表必须包含：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| tenant_id | varchar(64) | 租户，首期默认 `default` |
| created_at | timestamptz | 创建时间 |
| created_by | bigint | 创建人 |
| updated_at | timestamptz | 更新时间 |
| updated_by | bigint | 更新人 |
| deleted | boolean | 软删除 |
| version | int | 乐观锁或业务版本 |

## 3. 用户组织权限

### 3.1 sys_org

| 字段 | 类型 | 约束 |
| --- | --- | --- |
| id | bigint | pk |
| tenant_id | varchar(64) | not null |
| org_code | varchar(128) | unique |
| org_name | varchar(256) | not null |
| parent_id | bigint | nullable |
| status | varchar(32) | not null |

### 3.2 sys_user

| 字段 | 类型 | 约束 |
| --- | --- | --- |
| id | bigint | pk |
| tenant_id | varchar(64) | not null |
| username | varchar(128) | unique |
| display_name | varchar(256) | not null |
| email | varchar(256) | nullable |
| org_id | bigint | index |
| status | varchar(32) | not null |

### 3.3 sys_role / sys_permission / sys_user_role

用于 RBAC。权限编码按菜单、操作、数据范围和敏感字段权限分类。

## 4. 元模型与资产

### 4.1 meta_type

| 字段 | 类型 | 约束 |
| --- | --- | --- |
| id | bigint | pk |
| type_code | varchar(128) | unique |
| type_name | varchar(256) | not null |
| category | varchar(64) | not null |
| parent_type_code | varchar(128) | nullable |
| enabled | boolean | not null |

### 4.2 asset

| 字段 | 类型 | 约束 |
| --- | --- | --- |
| id | bigint | pk |
| tenant_id | varchar(64) | not null |
| asset_type | varchar(128) | not null, index |
| asset_code | varchar(256) | not null |
| asset_name | varchar(256) | not null |
| display_name | varchar(256) | nullable |
| description | text | nullable |
| business_domain | varchar(128) | index |
| system_code | varchar(128) | index |
| owner_user_id | bigint | index |
| steward_user_id | bigint | index |
| status | varchar(32) | index |
| source_type | varchar(32) | not null |
| deleted | boolean | not null |
| version | int | not null |

唯一约束：

```text
tenant_id + asset_type + asset_code + deleted
```

### 4.3 asset_attribute_value

| 字段 | 类型 | 约束 |
| --- | --- | --- |
| id | bigint | pk |
| tenant_id | varchar(64) | not null |
| asset_id | bigint | index |
| attr_code | varchar(128) | not null |
| attr_value | text | nullable |
| value_type | varchar(32) | not null |

## 5. 业务术语

### 5.1 business_term

| 字段 | 类型 | 约束 |
| --- | --- | --- |
| id | bigint | pk |
| tenant_id | varchar(64) | not null |
| term_code | varchar(128) | unique |
| term_name | varchar(256) | not null |
| english_name | varchar(256) | nullable |
| abbreviation | varchar(128) | nullable |
| definition | text | not null |
| scope | text | not null |
| business_domain | varchar(128) | index |
| owner_id | bigint | index |
| steward_id | bigint | index |
| status | varchar(32) | index |
| version | int | not null |

### 5.2 term_relation / term_reference / term_conflict

分别记录术语关系、术语引用和术语冲突。

关键索引：

- term_relation：from_term_id、to_term_id、relation_type。
- term_reference：term_id、asset_id、asset_type。
- term_conflict：conflict_type、status、owner_id。

## 6. 标准与引用

### 6.1 data_standard

保存数据标准、编码标准、指标标准等。

唯一约束：

```text
tenant_id + standard_code + deleted
```

### 6.2 standard_reference

连接数据标准和资产。

关键字段：

- standard_id。
- asset_id。
- asset_type。
- reference_type。
- status。

## 7. 血缘与关系

### 7.1 asset_relation

通用资产关系表。

### 7.2 lineage_relation

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| from_asset_id | bigint | 上游资产 |
| to_asset_id | bigint | 下游资产 |
| relation_type | varchar(64) | sync、transform、reference、derive、consume |
| lineage_level | varchar(64) | system、interface、table、field、report |
| source_type | varchar(32) | auto、manual、import、api |
| confirm_status | varchar(32) | pending、confirmed、need_fix、deprecated |
| confidence | varchar(32) | high、medium、low |

## 8. 自动化、任务、通知

核心表：

- governance_rule。
- rule_execution_log。
- collect_snapshot。
- diff_record。
- governance_issue。
- governance_task。
- notification。
- event_outbox。
- event_consume_log。

这些表支撑规则执行、差异识别、问题整改、通知和领域事件。

## 9. 工作流

核心表：

- workflow_instance。
- workflow_task。
- workflow_comment。
- workflow_attachment。

首期轻量实现审批任务模型，预留 workflow_engine_type 字段。

## 10. Flyway 策略

迁移脚本目录：

```text
backend/metadata-platform/src/main/resources/db/migration
```

命名：

```text
V1__init_security.sql
V2__init_metadata.sql
V3__init_glossary_standard.sql
V4__init_lineage.sql
V5__init_governance_automation.sql
```

规则：

- 已执行脚本禁止修改。
- 新变更只能新增版本脚本。
- 本地、测试、生产均通过 Flyway 初始化和升级。
