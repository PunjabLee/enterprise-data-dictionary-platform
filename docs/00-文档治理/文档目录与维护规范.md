# 文档目录与维护规范

## 1. 文档分级

| 等级 | 面向对象 | 典型文档 | 维护责任 |
| --- | --- | --- | --- |
| L1 总纲类 | 管理层、架构委员会、数据治理委员会 | 方案总纲、立项说明、路线图 | 项目牵头人、企业架构负责人 |
| L2 设计类 | 产品、架构、治理、实施负责人 | PRD、技术架构、元模型、治理流程 | 产品负责人、架构师、数据治理负责人 |
| L3 执行类 | 研发、测试、实施、系统团队 | 接口规范、采集规范、测试用例、实施计划 | 对应模块负责人 |
| L4 过程类 | 项目组 | 会议纪要、决策记录、问题台账 | PMO 或项目秘书 |
| L5 模板类 | 所有参与方 | 盘点表、评审表、审批表、验收表 | 数据治理团队、项目组 |

## 2. 命名规范

文档文件名建议采用：

`序号-主题-文档类型-v版本号.md`

示例：

- `01-技术架构设计-v0.1.md`
- `02-企业元模型设计说明书-v0.1.md`
- `03-数据血缘设计方案-v0.1.md`

已经形成正式名称的管理类文档，可以保留中文标题式命名，例如：

- `企业级数据字典管理系统方案总纲.md`
- `企业级数据字典管理系统PRD初稿.md`

## 3. 版本规则

| 版本 | 含义 |
| --- | --- |
| `v0.x` | 草稿、讨论稿、设计中 |
| `v1.0` | 首次评审通过，可作为实施依据 |
| `v1.x` | 小范围修订，不改变总体方向 |
| `v2.0` | 重大调整，例如范围、架构、流程或验收口径变化 |

## 4. 维护规则

- 每个目录必须保留 `README.md`，说明目录用途和当前文档状态。
- 所有正式文档必须进入 `00-文档治理/文档清单.md`。
- 关键决策必须记录到 `09-会议纪要与决策`，不能只停留在聊天记录或会议口头结论。
- 模板变更后，必须同步更新 `07-模板与表单` 的说明和生成脚本。
- 已废弃但有参考价值的文档进入 `10-归档`，不要直接删除。

## 5. 当前维护责任建议

| 文档类别 | 建议责任人 |
| --- | --- |
| 总纲与路线图 | 项目牵头人、企业架构负责人 |
| PRD 与页面需求 | 产品负责人 |
| 技术架构与接口 | 技术架构师 |
| 元模型与治理流程 | 数据架构师、数据治理负责人 |
| 盘点模板与表单 | Data Steward、项目组 |
| 测试验收 | 测试负责人、实施负责人 |
