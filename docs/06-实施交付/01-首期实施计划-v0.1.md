# 首期实施计划 v0.1

## 1. 实施目标

首期目标是在首批业务域内跑通数据架构治理闭环，形成可复制推广的方法、平台能力和运营机制。

## 2. 实施阶段

| 阶段 | 周期 | 目标 |
| --- | --- | --- |
| 阶段 1：准备 | 第 1 到 2 周 | 确认范围、角色、模板、资料清单 |
| 阶段 2：设计 | 第 2 到 4 周 | 完成详细 PRD、技术架构、元模型、流程设计 |
| 阶段 3：平台骨架 | 第 4 到 8 周 | 完成资产目录、字典、权限、导入、基础血缘 |
| 阶段 4：采集与治理 | 第 8 到 12 周 | 接入首批资产，跑通评审、认责、确认流程 |
| 阶段 5：验收推广 | 第 12 到 14 周 | 完成验收、复盘、推广计划 |

## 3. 首批交付物

- 详细 PRD。
- 技术架构设计。
- 元模型设计说明书。
- 权限与流程设计。
- 数据血缘设计方案。
- 采集适配设计。
- 首批资产盘点结果。
- 平台 MVP。
- 首期测试验收方案。
- 试点总结和推广建议。

## 4. 并行工作

资料等待期间可以推进：

- 平台工程骨架。
- 元模型和数据库初版设计。
- Mock 数据。
- Excel 导入。
- 资产目录和字典页面。
- 血缘图 PoC。
- 权限和审计基础能力。

## 5. 关键里程碑

| 里程碑 | 验收标准 |
| --- | --- |
| M1 文档体系完成 | 文档目录、规范、总纲、PRD、设计文档初稿齐备 |
| M2 元模型冻结 | 首期核心对象、属性、关系和状态确认 |
| M3 MVP 可运行 | 支持资产导入、查询、详情、字典维护 |
| M4 治理闭环跑通 | 支持架构评审、模型变更、责任认领、血缘确认 |
| M5 首批验收 | 首批业务域完成资产纳管和治理演示 |
