# 文档库首页

## 目录结构

| 目录 | 文档等级 | 用途 |
| --- | --- | --- |
| `00-文档治理` | 管理类 | 文档目录、命名规范、版本规则、维护机制 |
| `01-总体规划` | L1 总纲类 | 面向管理层、架构委员会、治理委员会的总体方案 |
| `02-产品需求` | L2 产品类 | PRD、用户故事、页面清单、原型说明、验收需求 |
| `03-架构设计` | L2/L3 技术类 | 技术架构、部署架构、模块设计、非功能设计 |
| `04-数据治理与元模型` | L2/L3 治理类 | 元模型、数据标准、术语、责任、分类分级、质量规则 |
| `05-接口与集成` | L3 集成类 | 采集适配、开放 API、外部系统集成、数据交换 |
| `06-实施交付` | 交付类 | 实施路线、里程碑、资源计划、风险清单、培训方案 |
| `07-模板与表单` | 模板类 | 盘点表、评审表、审批表、RACI、验收表 |
| `08-测试验收` | 测试类 | 测试方案、测试用例、验收标准、演示脚本 |
| `09-会议纪要与决策` | 过程类 | 会议纪要、决策记录、问题台账、变更记录 |
| `10-归档` | 归档类 | 废弃版本、历史材料、外部参考资料 |

## 推荐阅读顺序

1. 先看 `01-总体规划`，确认项目目标、范围和治理原则。
2. 再看 `02-产品需求`，确认用户角色、场景和功能范围。
3. 再看 `03-架构设计` 和 `04-数据治理与元模型`，确认系统如何建设。
4. 最后看 `06-实施交付`、`07-模板与表单` 和 `08-测试验收`，推进落地。
