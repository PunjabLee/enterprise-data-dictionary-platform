# 数据血缘设计方案 v0.1

## 1. 设计目标

首期血缘能力服务于架构治理、变更影响分析、报表溯源和责任追踪。血缘粒度固定为系统级、接口级、表级、报表级和关键字段级，不承诺字段级全量覆盖。

## 2. 血缘对象

| 对象 | 说明 |
| --- | --- |
| 系统 | 系统间数据交换和依赖 |
| 接口 | API、文件、消息、ESB 等交换关系 |
| 表 | 数据库表之间的同步、加工、引用 |
| 字段 | 首期仅覆盖关键字段 |
| ETL 作业 | 数据加工、同步和转换任务 |
| 数据集 | BI 或数据平台消费对象 |
| 报表 | 最终消费和展示对象 |
| 指标 | 业务口径和统计结果 |

## 3. 血缘关系类型

| 类型 | 说明 |
| --- | --- |
| 同步 | 数据从一个对象复制到另一个对象 |
| 加工 | 数据经过规则、SQL、脚本或任务处理 |
| 引用 | 下游对象直接引用上游对象 |
| 派生 | 字段或指标由其他字段计算生成 |
| 消费 | 报表、指标或应用消费数据集 |
| 发布 | 系统对外发布接口、文件或消息 |

## 4. 血缘属性

| 属性 | 说明 |
| --- | --- |
| 上游资产 | 血缘起点 |
| 下游资产 | 血缘终点 |
| 关系类型 | 同步、加工、引用、派生、消费、发布 |
| 粒度 | 系统、接口、表、字段、报表 |
| 生成方式 | 自动解析、人工维护、导入、API |
| 可信度 | 高、中、低 |
| 确认状态 | 待确认、已确认、需修正 |
| 责任人 | 血缘确认责任人 |
| 版本 | 血缘版本 |

## 5. 血缘生成方式

| 来源 | 首期策略 |
| --- | --- |
| 数据库视图 | 采集表、字段、视图依赖 |
| SQL 脚本 | 解析简单 SQL 的表级依赖 |
| ETL 配置 | 先支持导入和接口规范，后续做工具适配 |
| BI 元数据 | 先通过导入获取报表、数据集、来源表 |
| 接口文档 | 通过模板或 API 导入接口字段和系统依赖 |
| 人工确认 | 关键链路必须人工确认 |

## 6. 影响分析

支持从以下对象发起：

- 系统。
- 接口。
- 表。
- 关键字段。
- ETL 作业。
- 报表。
- 指标。

输出内容：

- 受影响资产。
- 影响路径。
- 影响层级。
- 影响类型。
- 责任人。
- 是否阻断变更。

## 7. 图展示策略

- 默认展示当前节点上下游一层。
- 支持按层级展开。
- 支持按资产类型过滤。
- 支持按已确认、待确认、低可信度过滤。
- 支持路径高亮。
- 支持导出影响清单。

## 8. 首期验收标准

- 核心报表能追溯到上游数据集、ETL、表和关键字段。
- 核心表变更能识别下游接口、ETL、报表、指标和责任人。
- 自动生成血缘可被人工确认和修正。
- 血缘确认结果可被审批流程引用。
