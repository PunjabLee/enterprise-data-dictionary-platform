# 工作流与审批任务设计 v0.1

## 1. 设计目标

首期采用轻量审批任务模型，支撑架构评审、模型变更、标准发布、血缘确认、责任认领、术语发布和例外豁免。后续流程复杂时可接入 Flowable。

## 2. 核心对象

| 对象 | 说明 |
| --- | --- |
| WorkflowInstance | 流程实例 |
| WorkflowTask | 审批或处理任务 |
| WorkflowComment | 审批意见 |
| WorkflowAttachment | 附件 |
| WorkflowAction | approve、reject、transfer、withdraw |

## 3. 流程类型

| 类型 | 说明 |
| --- | --- |
| architecture_review | 系统建设或改造架构评审 |
| model_change | 数据模型变更 |
| standard_publish | 数据标准发布 |
| term_publish | 业务术语发布 |
| lineage_confirm | 血缘确认 |
| responsibility_claim | 责任认领 |
| exception_waiver | 例外或豁免 |

## 4. 审批任务状态

| 状态 | 说明 |
| --- | --- |
| pending | 待处理 |
| processing | 处理中 |
| approved | 已同意 |
| rejected | 已驳回 |
| transferred | 已转办 |
| withdrawn | 已撤回 |
| canceled | 已取消 |

## 5. 流程实例状态

| 状态 | 说明 |
| --- | --- |
| draft | 草稿 |
| running | 运行中 |
| approved | 已通过 |
| rejected | 已驳回 |
| withdrawn | 已撤回 |
| closed | 已关闭 |

## 6. 审批路由

审批人来源：

- 固定角色。
- 业务域负责人。
- 系统 Owner。
- 数据 Owner。
- Data Steward。
- 安全合规人员。
- 规则引擎建议。

高风险场景：

- 影响敏感字段。
- 影响核心报表。
- 影响核心接口。
- 废止业务术语。
- 修改核心指标口径。

高风险场景必须增加会签或升级审批。

## 7. 与状态机关系

审批不替代业务对象状态机。

示例：

```text
模型变更审批通过
-> Model 状态从 评审中 到 已发布
-> 生成新版本
-> 发布 ModelChanged 事件
```

## 8. Flowable 预留

预留字段：

- workflow_engine_type。
- external_process_definition_id。
- external_process_instance_id。
- external_task_id。

首期默认：

```text
workflow_engine_type = lightweight
```
