# 状态机与版本规则设计 v0.1

## 1. 设计目标

状态机和版本规则用于保证资产、模型、标准、血缘和审批单在创建、确认、发布、变更、废弃过程中可控、可追溯。

## 2. 资产状态机

| 状态 | 说明 | 允许操作 |
| --- | --- | --- |
| 草稿 | 人工创建但未提交 | 编辑、删除、提交确认 |
| 待确认 | 采集、导入或提交后等待确认 | 确认、驳回、修正 |
| 已确认 | 责任人确认有效 | 发布、发起变更、废弃申请 |
| 已发布 | 正式资产目录可用 | 发起变更、废弃申请、查看版本 |
| 待修订 | 发现差异或问题 | 修订、提交审批 |
| 已废弃 | 不再作为有效资产 | 查看、恢复申请 |

## 3. 模型状态机

| 状态 | 说明 |
| --- | --- |
| 草稿 | 建模中 |
| 评审中 | 已提交模型评审 |
| 已驳回 | 评审不通过 |
| 已发布 | 正式模型版本 |
| 修订中 | 基于已发布版本修改 |
| 已废弃 | 模型不再使用 |

模型发布规则：

- 已发布模型不可直接编辑。
- 变更必须生成新版本。
- 新版本发布后，旧版本进入历史状态。
- 模型版本必须关联审批记录。

## 4. 数据标准状态机

| 状态 | 说明 |
| --- | --- |
| 草稿 | 标准起草中 |
| 评审中 | 已提交评审 |
| 已发布 | 可被资产引用 |
| 修订中 | 对已发布标准进行修改 |
| 已废止 | 不再允许新增引用 |

标准引用规则：

- 只有已发布标准可以被正式引用。
- 已废止标准保留历史引用。
- 标准修订后应生成受影响资产清单。

## 5. 血缘状态机

| 状态 | 说明 |
| --- | --- |
| 待确认 | 自动解析、导入或人工创建后未确认 |
| 已确认 | 责任人确认可用 |
| 需修正 | 发现错误，等待修正 |
| 已废弃 | 血缘关系不再有效 |

血缘使用规则：

- 默认影响分析只使用已确认血缘。
- 用户可选择包含待确认血缘，但报告需标识可信度。
- 关键字段级血缘必须经过人工确认。

## 6. 审批单状态机

| 状态 | 说明 |
| --- | --- |
| 草稿 | 未提交 |
| 审批中 | 流程进行中 |
| 已通过 | 审批完成且通过 |
| 已驳回 | 审批不通过 |
| 已撤回 | 提交人撤回 |
| 已关闭 | 流程终止或归档 |

审批规则：

- 审批中对象原则上禁止直接修改，需撤回或补充材料。
- 审批通过后触发发布或状态变更。
- 审批意见、节点、处理人和时间必须记录。

## 7. 版本号规则

- 草稿内部修改不递增正式版本。
- 首次发布版本为 `1`。
- 每次发布递增主版本整数。
- 小修订说明记录在版本快照中，不单独使用语义化版本。
- 版本快照必须保存核心属性、扩展属性和关系摘要。
