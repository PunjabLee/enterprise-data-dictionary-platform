# 通知中心与任务中心设计 v0.1

## 1. 设计目标

通知中心与任务中心用于承接平台自动化引擎、治理流程和人工协作产生的待办、提醒、整改、确认、审批和订阅消息，是治理闭环能否持续运营的关键能力。

## 2. 任务中心

任务类型：

| 类型 | 说明 |
| --- | --- |
| approval | 审批任务 |
| confirmation | 确认任务 |
| remediation | 整改任务 |
| responsibility | 认责任务 |
| review | 架构评审任务 |
| exception | 例外处理任务 |
| import_fix | 导入错误修正任务 |
| lineage_confirm | 血缘确认任务 |

任务属性：

| 属性 | 说明 |
| --- | --- |
| taskNo | 任务编号 |
| taskType | 任务类型 |
| title | 标题 |
| description | 描述 |
| assigneeId | 处理人 |
| relatedObjectType | 关联对象类型 |
| relatedObjectId | 关联对象 ID |
| priority | 优先级 |
| dueDate | 截止日期 |
| status | 状态 |
| sourceType | 来源 |

## 3. 任务状态

| 状态 | 说明 |
| --- | --- |

| processing | 处理中 |

## 4. 通知中心

通知类型：

| 类型 | 说明 |
| --- | --- |
| task_assigned | 任务分派 |
| approval_required | 审批待办 |
| issue_created | 问题生成 |
| remediation_due | 整改到期 |
| asset_changed | 关注资产变更 |
| collect_failed | 采集失败 |
| import_completed | 导入完成 |
| exception_expiring | 例外即将到期 |
| report_ready | 报告生成完成 |

通知渠道：

- 站内通知。
- 邮件。
- 企业微信。
- 钉钉。
- 外部待办平台。

首期必须支持站内通知，其他渠道通过适配器预留。

## 6. 任务与通知关系

```text
Task Created
-> Notification Created
-> Receiver Reads Notification
-> Receiver Opens Task
-> Task Completed
-> Notification Closed
```

一个任务可以产生多个通知，一个通知通常关联一个任务或一个资产变更。

## 7. 订阅与关注

用户可关注：

- 系统。
- 表。
- 字段。
- 报表。
- 指标。
- 业务术语。
- 数据标准。
- 血缘关系。

触发通知：

- 资产发布。
- 资产变更。
- 资产废弃。
- 术语修订。
- 标准发布。
- 血缘确认。
- 影响分析命中关注资产。

## 8. 发送策略

- 高优先级任务立即通知。
- 普通提醒可聚合发送。
- 重复通知应合并。
- 发送失败应重试。
- 通知模板可配置。

## 9. 权限与审计

- 用户只能查看自己的任务和通知。
- 管理员可按授权范围查看任务统计。
- 任务转办、关闭、取消必须审计。
- 敏感资产相关通知不得泄露敏感字段明文。
