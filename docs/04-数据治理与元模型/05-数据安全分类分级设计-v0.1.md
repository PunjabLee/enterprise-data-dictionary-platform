# 数据安全分类分级设计 v0.1

## 1. 设计目标

数据安全分类分级用于识别敏感数据、控制展示和导出权限、支撑审计和安全合规。

## 2. 分类

建议分类：

| 分类 | 说明 |
| --- | --- |
| public | 公开数据 |
| internal | 内部数据 |
| sensitive | 敏感数据 |
| personal | 个人信息 |
| important | 重要数据 |
| secret | 受限或涉密数据 |

## 3. 分级

建议分级：

| 等级 | 说明 |
| --- | --- |
| L1 | 公开或低敏 |
| L2 | 内部使用 |
| L3 | 敏感，需控制访问 |
| L4 | 高敏，需审批查看或导出 |
| L5 | 严格受限，默认不可导出 |

## 4. 敏感识别规则

识别依据：

- 字段名。
- 字段中文名。
- 业务定义。
- 数据类型。
- 数据标准。
- 业务术语。
- 正则规则。

示例：

- 手机号。
- 身份证号。
- 银行卡号。
- 地址。
- 邮箱。
- 账户余额。
- 风险评级。

## 5. 展示策略

| 等级 | 默认展示 |
| --- | --- |
| L1 | 完整展示 |
| L2 | 完整展示 |
| L3 | 部分字段脱敏 |
| L4 | 默认隐藏敏感说明，需要权限 |
| L5 | 默认不可见，需要专项授权 |

注意：本平台主要管理元数据，不查询业务明细数据；脱敏主要针对字段定义、样例、枚举、备注和导出内容。

## 6. 导出控制

- L3 及以上资产导出必须审计。
- L4 导出需要导出权限。
- L5 默认禁止导出。
- 导出文件应记录导出人、时间、条件、字段、原因。

## 7. 审计要求

必须审计：

- 查看敏感字段详情。
- 修改分类分级。
- 导出包含敏感字段的资产。
- 授权敏感字段权限。
- 规则自动识别结果被人工覆盖。

## 8. 自动化

自动化能力：

- 敏感字段自动识别建议。
- 分类分级自动建议。
- 未分级敏感字段生成整改任务。
- 分类分级变更触发影响分析。

自动化边界：

- 自动识别只是建议。
- L3 及以上分级必须人工确认。
- 安全合规角色拥有最终确认权。
