# 业务术语管理设计 v0.1

## 1. 设计定位

业务术语管理是企业数据治理的语义中枢，用于统一企业内部业务语言，连接业务概念、数据标准、指标口径、数据模型、数据字典和数据资产。

业务术语不应被简单归入数据标准。二者关系如下：

| 对象 | 重点 | 示例 |
| --- | --- | --- |
| 业务术语 | 统一业务语言和概念含义 | 客户、有效客户、开户日期、风险等级 |
| 数据标准 | 统一数据表达、格式、命名和约束 | 客户编号格式、日期格式、证件类型编码 |
| 指标口径 | 统一统计计算规则 | 月活客户数、逾期率、销售转化率 |
| 数据字典 | 承载字段级业务含义和技术属性 | `customer_id` 字段定义 |
| 数据模型 | 承载业务概念、逻辑结构和物理实现 | 客户实体、订单实体、客户表 |

## 2. 建设目标

- 建立企业统一业务术语库。
- 消除同名不同义、同义不同名、缩写混乱和跨部门口径冲突。
- 支撑字段、指标、报表、接口、模型对象引用权威术语。
- 支撑业务术语变更影响分析。
- 支撑业务术语审批、发布、废止和版本管理。
- 支撑术语图谱和术语到数据资产的映射。

## 3. 核心功能

| 功能 | 说明 |
| --- | --- |
| 术语目录 | 按业务域、主题域、部门、数据域组织术语 |
| 术语维护 | 维护名称、英文名、缩写、别名、同义词、定义、适用范围 |
| 术语关系 | 维护同义、包含、派生、关联、上位词、下位词 |
| 术语责任 | 维护 Owner、Steward、审批人、归属部门 |
| 术语生命周期 | 草稿、评审中、已发布、修订中、已废止 |
| 术语审批 | 新增、修订、废止走审批流程 |
| 术语引用 | 字段、逻辑属性、指标、报表、接口字段、标准引用术语 |
| 术语冲突检查 | 检查同名不同义、同义不同名、缩写冲突 |
| 术语影响分析 | 分析术语变更影响哪些字段、指标、报表、模型 |
| 术语推荐 | 字段、指标、模型维护时自动推荐可引用术语 |
| 术语图谱 | 展示术语关系和术语到资产的映射 |
| 导入导出 | 支持术语批量初始化和治理结果导出 |
| 治理指标 | 术语引用率、冲突数、待审批数、废止影响数 |

## 4. 术语属性

| 属性 | 说明 | 是否必填 |
| --- | --- | --- |
| 术语编码 | 全局唯一编码 | 是 |
| 术语名称 | 中文名称 | 是 |
| 英文名称 | 英文名 | 否 |
| 缩写 | 常用缩写 | 否 |
| 别名 | 其他名称 | 否 |
| 同义词 | 语义相同的其他术语 | 否 |
| 业务定义 | 权威业务解释 | 是 |
| 适用范围 | 业务域、系统、部门或场景 | 是 |
| 不适用范围 | 明确排除场景 | 否 |
| 示例 | 业务示例 | 否 |
| 归属业务域 | 所属业务域 | 是 |
| 归属部门 | 负责部门 | 是 |
| 术语 Owner | 业务责任人 | 是 |
| Data Steward | 维护人 | 是 |
| 状态 | 生命周期状态 | 是 |
| 版本 | 当前版本 | 是 |

## 5. 术语关系

| 关系 | 说明 | 示例 |
| --- | --- | --- |
| 同义 | 两个术语语义相同 | 客户 / 客户主体 |
| 近义 | 语义接近但不完全相同 | 活跃客户 / 有效客户 |
| 包含 | 一个术语包含另一个术语 | 客户 包含 个人客户、企业客户 |
| 上下位 | 术语分类层级 | 客户 是 个人客户 的上位词 |
| 派生 | 一个术语由另一个术语派生 | 有效客户 派生自 客户 |
| 关联 | 存在业务关联 | 客户 与 账户 |
| 引用 | 数据资产引用术语 | 字段 `customer_id` 引用 客户 |

## 6. 生命周期

| 状态 | 说明 | 允许操作 |
| --- | --- | --- |
| 草稿 | 术语创建但未提交 | 编辑、删除、提交评审 |
| 评审中 | 已提交审批 | 审批、驳回、撤回 |
| 已发布 | 可被正式引用 | 发起修订、发起废止、影响分析 |
| 修订中 | 基于已发布版本修改 | 编辑、提交评审 |
| 已废止 | 不再允许新增引用 | 查看历史、影响追踪 |

规则：

- 只有已发布术语可被正式引用。
- 已废止术语保留历史引用，不允许新增引用。
- 术语修订必须生成新版本。
- 术语废止必须执行影响分析。

## 7. 术语审批

审批场景：

- 新增术语。
- 修订术语定义。
- 合并术语。
- 拆分术语。
- 废止术语。

审批角色：

- 术语 Owner。
- Data Steward。
- 数据治理管理员。
- 业务域负责人。
- 必要时数据架构师参与会签。

审批检查项：

- 是否与已有术语冲突。
- 定义是否清晰。
- 适用范围是否明确。
- 是否存在同义词或别名。
- 是否影响已发布字段、指标、报表或模型。

## 8. 术语引用

可引用术语的对象：

- 字段。
- 逻辑属性。
- 概念实体。
- 逻辑实体。
- 指标。
- 报表。
- 接口字段。
- 数据标准。
- 业务规则。

引用规则：

- 一个资产可引用多个术语。
- 一个术语可被多个资产引用。
- 核心字段和核心指标应优先引用已发布术语。
- 引用废止术语的资产应进入整改清单。

## 9. 术语冲突检查

冲突类型：

| 类型 | 说明 |
| --- | --- |
| 同名不同义 | 名称相同但定义不同 |
| 同义不同名 | 定义相同或高度相似但名称不同 |
| 缩写冲突 | 同一缩写对应多个术语 |
| 跨域冲突 | 不同业务域对同一术语定义不一致 |
| 引用冲突 | 字段或指标引用了不适用的术语 |

处理方式：

- 生成冲突问题。
- 指定责任人。
- 发起合并、拆分或定义澄清。
- 记录处理结论。

## 10. 术语推荐

推荐触发场景：

- 字段业务定义维护。
- 逻辑属性维护。
- 指标口径维护。
- 报表说明维护。
- 接口字段说明维护。

推荐依据：

- 名称匹配。
- 英文名匹配。
- 缩写匹配。
- 别名匹配。
- 业务定义相似度。
- 所属业务域。

首期可采用规则和关键词匹配，后续可引入向量检索或语义检索。

## 11. 术语影响分析

术语变更需分析：

- 引用该术语的字段。
- 引用该术语的逻辑属性。
- 引用该术语的指标。
- 引用该术语的报表。
- 引用该术语的数据标准。
- 相关概念模型和逻辑模型。

输出：

- 受影响资产清单。
- 责任人。
- 影响等级。
- 是否需要整改。
- 是否阻断术语发布或废止。

## 12. 自动化能力

| 自动化能力 | 说明 |
| --- | --- |
| 术语冲突自动识别 | 自动识别同名、同义、缩写冲突 |
| 术语引用自动推荐 | 根据字段名、指标名、定义推荐术语 |
| 废止影响自动分析 | 废止术语前自动生成影响清单 |
| 术语整改任务自动生成 | 对引用废止术语或冲突术语的资产生成整改任务 |
| 术语治理指标自动统计 | 自动统计引用率、冲突数、待审批数 |

| 对象 | 说明 |
| --- | --- |

```text
BusinessTerm -> TermRelation -> BusinessTerm
BusinessTerm -> TermReference -> Asset
BusinessTerm -> TermConflict -> BusinessTerm
BusinessTerm -> TermVersion
```
