# 企业元模型设计说明书 v0.1

## 1. 设计目标

元模型用于统一描述企业数据资产、数据模型、数据标准、血缘关系、责任关系和治理流程对象。首期目标是支持架构治理闭环，而不是追求理论上的全量元数据覆盖。

## 2. 元模型分层

| 层次 | 对象 |
| --- | --- |
| 架构层 | 业务域、数据域、系统、应用、模块、架构视图 |
| 数据层 | 数据库、Schema、表、字段、文件、消息 Topic |
| 模型层 | 概念实体、逻辑实体、逻辑属性、物理表、物理字段 |
| 标准层 | 数据标准、业务术语、指标标准、编码标准、参考数据 |
| 集成层 | 接口、API、文件交换、ETL 作业、调度任务、数据集 |
| 消费层 | 报表、看板、指标、分析主题 |
| 治理层 | 责任关系、审批单、评审记录、问题单、例外申请 |
| 关系层 | 包含、引用、映射、依赖、血缘、责任、影响 |

## 3. 核心对象

### 3.1 系统

必填属性：

- 系统名称。
- 系统编码。
- 所属业务域。
- 系统 Owner。
- 技术负责人。
- 生命周期状态。

关键关系：

- 系统包含数据库、接口、报表、ETL、数据集。
- 系统向其他系统提供或消费接口。
- 系统拥有数据 Owner 和技术负责人。

### 3.2 表

必填属性：

- 表名。
- 表中文名。
- 所属系统。
- 所属数据库和 Schema。
- 业务说明。
- 是否核心表。
- 数据 Owner。
- Data Steward。
- 状态。

关键关系：

- 表包含字段。
- 表映射逻辑实体。
- 表参与血缘关系。
- 表引用数据标准或业务术语。

### 3.3 字段

必填属性：

- 字段名。
- 字段中文名。
- 数据类型。
- 业务定义。
- 是否关键字段。
- 是否敏感。
- 分类分级。
- 标准映射。
- 责任人。

关键关系：

- 字段属于表、接口、文件、报表或数据集。
- 字段映射逻辑属性。
- 字段引用数据标准。
- 关键字段参与字段级血缘。

### 3.4 数据标准

必填属性：

- 标准名称。
- 标准编码。
- 标准类型。
- 定义。
- 数据格式。
- 适用范围。
- 责任部门。
- 状态。

关键关系：

- 标准被字段、逻辑属性、接口字段、指标引用。
- 标准发布后产生待整改资产清单。

### 3.5 血缘关系

必填属性：

- 上游资产。
- 下游资产。
- 关系类型。
- 血缘粒度。
- 来源方式。
- 确认状态。
- 可信度。

关系类型：

- 同步。
- 加工。
- 引用。
- 派生。
- 消费。
- 发布。

## 4. 通用属性

所有核心对象建议具备：

- `id`：全局唯一标识。
- `code`：业务编码或技术编码。
- `name`：名称。
- `displayName`：展示名称。
- `description`：说明。
- `status`：状态。
- `version`：版本。
- `sourceType`：采集、人工、导入、API。
- `createdBy`、`createdAt`、`updatedBy`、`updatedAt`。

## 5. 状态模型

| 状态 | 含义 |
| --- | --- |
| 待采集 | 已定义采集范围，但尚未采集 |
| 待确认 | 已采集或导入，等待责任人确认 |
| 已确认 | 责任人确认信息有效 |
| 已发布 | 进入正式资产目录，可用于治理流程 |
| 待修订 | 已发现变化或问题，等待修订 |
| 已废弃 | 不再作为当前有效资产 |

## 6. 版本规则

- 草稿对象允许编辑。
- 已发布对象变更必须生成新版本。
- 历史版本只读。
- 审批记录必须关联资产版本。
- 采集快照与发布版本分开保存。

## 7. 首期元模型边界

首期必须实现：

- 系统、数据库、Schema、表、字段。
- 接口、ETL、调度、报表、指标。
- 概念实体、逻辑实体、逻辑属性、物理映射。
- 数据标准、业务术语。
- 责任关系、血缘关系、审批单。

首期暂不强制实现：

- 复杂主数据分发模型。
- 实时数据流作业完整血缘。
- 字段级全量血缘。
- 全套 TOGAF 架构制品库。
