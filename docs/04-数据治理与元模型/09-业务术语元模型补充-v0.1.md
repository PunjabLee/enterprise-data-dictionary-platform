# 业务术语元模型补充 v0.1

## 1. 设计目标

本文档补充 Business Glossary 相关元模型，明确业务术语、术语关系、术语引用、术语冲突和术语版本在统一元模型中的表达方式。

## 2. 核心对象

| 对象 | 说明 |
| --- | --- |
| BusinessTerm | 单个业务术语 |
| BusinessGlossary | 业务术语库或术语目录 |
| TermRelation | 术语之间的语义关系 |
| TermReference | 术语与资产之间的引用关系 |
| TermConflict | 术语冲突记录 |
| TermVersion | 术语版本快照 |

## 3. BusinessTerm

核心属性：

| 属性 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| termCode | string | 是 | 术语编码，全局唯一 |
| termName | string | 是 | 中文名称 |
| englishName | string | 否 | 英文名称 |
| abbreviation | string | 否 | 缩写 |
| aliases | string[] | 否 | 别名 |
| synonyms | string[] | 否 | 同义词 |
| definition | text | 是 | 业务定义 |
| scope | text | 是 | 适用范围 |
| excludedScope | text | 否 | 不适用范围 |
| example | text | 否 | 示例 |
| businessDomain | string | 是 | 归属业务域 |
| departmentId | string | 是 | 归属部门 |
| ownerId | string | 是 | 术语 Owner |
| stewardId | string | 是 | Data Steward |
| status | enum | 是 | 生命周期状态 |
| version | int | 是 | 当前版本 |

## 7. TermVersion

版本快照内容：

- 术语基础属性。
- 术语关系摘要。
- 术语引用摘要。
- 变更说明。
- 审批单 ID。
- 发布人。
- 发布时间。

规则：

- 已发布术语不可直接修改。
- 修订必须生成新版本。
- 废止术语必须保留历史引用。
- 术语版本必须关联审批记录。

## 8. 与统一资产模型的关系

BusinessTerm 可作为资产类型纳入统一 `asset` 表，也可作为独立业务表管理。首期建议：

- `BusinessTerm` 作为独立领域对象。
- 同步生成统一资产索引，供资产目录检索。
- 通过 `TermReference` 连接术语和资产。

这样可以兼顾术语管理的专业性和资产目录的统一检索。

## 9. 自动化规则

业务术语相关规则：

- 核心字段应引用已发布术语。
- 指标应引用至少一个业务术语。
- 术语废止前必须完成影响分析。
- 同一业务域内不允许同名不同义术语直接发布。
- 缩写冲突必须生成治理问题。
