# 问题台账与整改闭环设计 v0.1

## 1. 设计目标

问题台账与整改闭环用于把平台发现的治理问题转化为可跟踪、可分派、可整改、可验收的任务，避免平台只发现问题但无法推动治理落地。

## 2. 问题来源

| 来源 | 示例 |
| --- | --- |
| 规则校验 | 定义缺失、标准未引用、命名不合规 |
| 采集差异 | 字段删除、类型变化、表新增 |
| 血缘分析 | 血缘断点、低可信度血缘、未确认血缘 |
| 术语治理 | 同名不同义、缩写冲突、引用废止术语 |
| 审批流程 | 架构评审驳回、模型变更风险 |
| 安全检查 | 敏感字段未分级、导出权限异常 |
| 人工登记 | 架构师、Owner、Steward 手工提交问题 |

## 3. 问题类型

| 类型 | 说明 |
| --- | --- |
| metadata_missing | 元数据缺失 |
| standard_violation | 标准不合规 |
| glossary_conflict | 术语冲突 |
| responsibility_missing | 责任人缺失 |
| lineage_break | 血缘断点 |
| security_risk | 安全风险 |
| architecture_exception | 架构例外 |
| import_error | 导入错误 |
| collect_conflict | 采集冲突 |

## 4. 问题属性

| 属性 | 说明 |
| --- | --- |
| issueNo | 问题编号 |
| issueType | 问题类型 |
| severity | 严重等级 |
| sourceType | 来源类型 |
| relatedAssetId | 关联资产 |
| relatedRuleId | 关联规则 |
| description | 问题说明 |
| ownerId | 整改责任人 |
| dueDate | 截止时间 |
| status | 状态 |
| resolution | 处理结论 |

## 5. 问题状态

| 状态 | 说明 |
| --- | --- |
| open | 已创建 |
| assigned | 已分派 |
| processing | 处理中 |
| resolved | 已处理 |
| verified | 已验证 |
| closed | 已关闭 |
| waived | 已豁免 |
| reopened | 重新打开 |

## 6. 整改闭环

```text
发现问题
-> 生成问题
-> 分派责任人
-> 整改处理
-> 提交验证
-> 规则复查或人工验收
-> 关闭问题
```

## 7. 自动生成规则

自动生成问题的场景：

- 核心资产缺少 Owner。
- 核心字段缺少业务定义。
- 核心字段未引用标准。
- 字段命中敏感规则但未分级。
- 血缘关系长期未确认。
- 术语存在同名不同义冲突。
- 采集发现已发布字段被删除。

## 8. 整改任务

整改任务与问题一一关联或一对多关联。

任务字段：

- taskNo。
- issueId。
- assigneeId。
- priority。
- dueDate。
- status。
- actionResult。
- attachments。

## 9. 验收与关闭

关闭条件：

- 自动规则复查通过。
- 责任人提交整改说明。
- 审批人或治理人员确认。
- 相关资产状态更新。

豁免条件：

- 必须填写豁免原因。
- 必须指定豁免有效期。
- 必须指定豁免审批人。
- 到期自动重新打开或提醒。

## 10. 看板指标

- 未关闭问题数。
- 超期问题数。
- 高严重等级问题数。
- 按业务域统计问题。
- 按系统统计问题。
- 按责任人统计问题。
- 整改及时率。
- 问题复发率。
